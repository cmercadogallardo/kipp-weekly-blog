📋 Resumen Ejecutivo

ACP (Agent Client Protocol) es el “LSP para agentes de IA” — un estándar abierto JSON-RPC que desacopla editores de código de los agentes de IA que los impulsan.

Hallazgo clave: Los 4 principales coding agents (Gemini CLI, Claude Code, Codex CLI, OpenCode) tienen estrategias DIFERENTES de integración ACP/MCP, y cada uno ofrece ventajas únicas que podemos explotar desde OpenClaw.

🎯 ¿Qué es ACP? (Agent Client Protocol)

Definición

ACP estandariza la comunicación entre editores de código (IDEs) y agentes de coding con IA. Es análogo al Language Server Protocol (LSP) que revolucionó los editores de código hace una década.

El Problema que Resuelve

Antes de ACP:

Editor × Agente = Integración personalizada
- VS Code × Claude Code → Integración custom
- VS Code × Codex → Integración custom  
- Zed × Gemini CLI → Integración custom
- JetBrains × Copilot → Integración custom

Resultado: N × M integraciones bespoke

Con ACP:

Editor (ACP Client) ←→ ACP Protocol ←→ Agente (ACP Agent)
- Cualquier editor ACP-compatible
- Cualquier agente ACP-compatible
- UNA sola integración estándar

Resultado: N + M integraciones

Analogía LSP

LSP (Language Server Protocol) ACP (Agent Client Protocol)
Desacopla lenguaje ↔︎ editor Desacopla agente ↔︎ editor
Un servidor de lenguaje funciona en cualquier editor Un agente funciona en cualquier editor
Microsoft estandarizó (2016) Zed/Google estandarizaron (2025)
JSON-RPC sobre stdio/HTTP JSON-RPC sobre stdio/HTTP

🔍 Estado de ACP por CLI

1️⃣ Gemini CLI (Google) — 🟢 NATIVO

Estado: Implementación de referencia del protocolo ACP

Comando:

gemini --acp

Características: - ✅ ACP nativo desde el core (no adapter) - ✅ JSON-RPC 2.0 sobre stdio - ✅ Soporte MCP integrado (puede conectar a MCP servers) - ✅ File system proxy (seguridad: el cliente controla acceso a archivos) - ✅ Métodos soportados: - initialize — Handshake inicial + registro MCP - authenticate — Autenticación de usuario - newSession — Nueva sesión de chat - loadSession — Cargar sesión previa - prompt — Enviar prompt al agente - cancel — Cancelar prompt en curso - setSessionMode — Cambiar nivel de aprobación (auto-approve, etc.) - unstable_setSessionModel — Cambiar modelo dinámicamente

Arquitectura:

Editor (ACP Client)
       ↓ JSON-RPC stdio
Gemini CLI (ACP Server + MCP Client)
       ↓ MCP
MCP Servers (herramientas externas)

Ventajas: - Implementación más madura y completa - Documentación oficial exhaustiva - File system proxy para seguridad - Telemetría integrada para debugging

Desventajas: - Modelo Gemini solamente (obvio) - Menos flexible que adapters custom

Repos/Links: - https://github.com/google-gemini/gemini-cli - https://github.com/google-gemini/gemini-cli/blob/main/docs/cli/acp-mode.md - https://agentclientprotocol.com/

2️⃣ Claude Code (Anthropic) — 🟡 ADAPTER

Estado: ACP vía adapter oficial (claude-agent-acp)

Comando:

# No tiene flag --acp nativo
# Se usa el adapter de Zed:
npx @zed-industries/claude-code-acp

Características: - 🟡 Requiere adapter (no nativo en CLI) - ✅ Adapter oficial mantenido por Zed/Anthropic - ✅ MCP nativo en Claude Code CLI (claude mcp add) - ✅ Soporta 3 scopes: local, project, user - ✅ Canales para push messages (eventos externos) - ✅ Plugin system con MCP servers embebidos

Arquitectura:

Editor (ACP Client)
       ↓ JSON-RPC stdio
claude-agent-acp (Adapter/Translator)
       ↓ SDK de Claude Code
Claude Code CLI (MCP Client)
       ↓ MCP
MCP Servers

MCP en Claude Code:

# Agregar servidor MCP
claude mcp add --transport http stripe https://mcp.stripe.com
claude mcp add --transport stdio airtable -- npx -y airtable-mcp-server

# Ver servidores
claude mcp list

# Dentro de Claude Code
/mcp

Ventajas: - MCP más maduro y documentado - Plugin system con MCP embebido - Canales para eventos push (CI, monitoring, chat) - Gestión de scopes (local/project/user)

Desventajas: - Requiere adapter (no nativo) - Adapter mantenido por terceros (Zed)

Repos/Links: - https://github.com/zed-industries/claude-code-acp - https://docs.anthropic.com/en/docs/claude-code/mcp/ - https://www.npmjs.com/package/@zed-industries/claude-code-acp

3️⃣ Codex CLI (OpenAI) — 🟡 APP-SERVER PROPIO

Estado: Protocolo propietario similar a ACP (app-server)

Comando:

codex app-server --listen stdio://

Características: - 🟡 No usa ACP directamente (protocolo propietario) - ✅ JSON-RPC 2.0 (similar a ACP pero no compatible) - ✅ app-server para integraciones ricas (VS Code extension) - ✅ Primitivas: Thread, Turn, Item - ✅ Existe adapter: codex-acp para compatibilidad ACP - ✅ MCP experimental (codex-rs/docs/codex_mcp_interface.md)

Arquitectura:

Editor (VS Code, etc.)
       ↓ JSON-RPC stdio
Codex App Server
       ↓
Codex CLI (MCP Client experimental)
       ↓ MCP
MCP Servers

Primitivas del App Server:

Primitiva Descripción Operaciones
Thread Conversación usuario-agente thread/start, thread/resume, thread/fork, thread/list, thread/archive
Turn Un ciclo prompt-respuesta turn/start, turn/interrupt, turn/steer
Item Mensajes y acciones userMessage, agentMessage, reasoning, commandExecution, fileChange, mcpToolCall, webSearch

Adapter ACP:

# Para usar Codex desde editores ACP:
# https://github.com/zed-industries/codex-acp

Ventajas: - App Server muy bien diseñado - Primitivas claras (Thread/Turn/Item) - VS Code extension oficial usa esto - Adapter ACP disponible

Desventajas: - Protocolo propietario (no ACP nativo) - MCP aún experimental - Requiere adapter para ACP

Repos/Links: - https://github.com/openai/codex - https://developers.openai.com/codex/app-server - https://openai-codex.mintlify.app/api/overview - https://github.com/zed-industries/codex-acp

4️⃣ OpenCode — 🟢 NATIVO + MCP + ACP

Estado: Soporte dual MCP + ACP nativos

Comando:

# MCP (ya configurado)
~/.config/opencode/opencode.jsonc

# ACP (si está disponible en tu versión)
opencode --acp

Características: - ✅ MCP nativo configurado vía JSON - ✅ ACP reportado en investigación previa - ✅ GitHub Copilot como provider principal - ✅ Múltiples modelos (GPT-5.x, Claude Sonnet, Gemini) - ✅ Agent modes (plan, code, etc.)

Arquitectura:

Editor (ACP Client)
       ↓ JSON-RPC stdio
OpenCode (ACP Server + MCP Client)
       ↓ MCP
MCP Servers
       ↓
GitHub Copilot API (u otros providers)

Providers disponibles: - github-copilot/gpt-5.2-codex - github-copilot/gpt-5.3-codex - github-copilot/gpt-5.4 - github-copilot/claude-sonnet-4.5 - github-copilot/gpt-4o - github-copilot/gemini-2.5-pro

Ventajas: - Multi-modelo (puedes elegir) - MCP + ACP nativos - Integración directa con Copilot (suscripción existente) - Menos vendor lock-in

Desventajas: - Documentación menos madura - Comunidad más pequeña

Repos/Links: - https://github.com/OpenCodeDev/opencode - skills/opencode-github-copilot/SKILL.md (local)

🧠 Conceptos Clave

ACP vs MCP: Complementarios, No Competidores

MCP ACP
Entre: Agente ↔︎ Herramientas Entre: Editor ↔︎ Agente
Problema: ¿Cómo llama el agente a herramientas? Problema: ¿Cómo controla el editor al agente?
Cliente: Aplicación de IA / host Cliente: Editor de código / IDE
Servidor: Proveedor de herramientas Subproceso: Agente de coding AI

Shorthand: > MCP le da herramientas al agente.
> ACP le da un editor al agente.

Arquitectura Típica

┌─────────────────────────────────────────────────────────┐
│                    EDITOR (ACP Client)                  │
│  - VS Code, Zed, JetBrains, Neovim, Emacs              │
│  - Renderiza chat, diffs, permission prompts           │
│  - Inicia conexión ACP                                 │
└────────────────────┬────────────────────────────────────┘
                     │ JSON-RPC 2.0 (stdio/HTTP)
                     │ ACP Protocol
                     ▼
┌─────────────────────────────────────────────────────────┐
│                  AGENTE (ACP Server)                    │
│  - Gemini CLI, Claude Code, Codex, OpenCode            │
│  - Ejecuta LLM loop                                    │
│  - Llama herramientas vía MCP                          │
│  - También es MCP Client                               │
└────────────────────┬────────────────────────────────────┘
                     │ MCP Protocol (stdio/HTTP/SSE)
                     ▼
┌─────────────────────────────────────────────────────────┐
│                   MCP SERVERS                           │
│  - GitHub, GitLab, Jira, Slack, PostgreSQL, etc.       │
│  - Herramientas y fuentes de datos externas            │
└─────────────────────────────────────────────────────────┘

Direccionalidad (¡Cuidado con la confusión!)

En MCP: - La aplicación de IA es el HOST/CLIENT - Las herramientas son SERVERS

En ACP: - El editor es el CLIENT - El agente es el SERVER/SUBPROCESO

Los términos se repiten pero los roles se invierten. ⚠️

💡 Harness Differences por CLI

Cada CLI tiene un “harness” diferente — la forma en que encapsula el agente y expone controles.

Gemini CLI Harness

Tipo: ACP Server nativo

# Modo ACP
gemini --acp

# El agente:
# 1. Escucha JSON-RPC en stdio
# 2. Procesa requests (initialize, prompt, cancel, etc.)
# 3. Notifica eventos (fileChange, commandExecution, etc.)
# 4. Conecta a MCP servers si se proporcionan

Ventaja: Diseño limpio desde el inicio con ACP en mente.

Claude Code Harness

Tipo: CLI tradicional + Adapter ACP

# CLI normal (no ACP)
claude "haz algo"

# ACP vía adapter
npx @zed-industries/claude-code-acp

# El adapter:
# 1. Recibe JSON-RPC ACP del editor
# 2. Traduce a llamadas al SDK de Claude Code
# 3. Traduce respuestas de Claude Code a JSON-RPC ACP

Ventaja: MCP más maduro, plugin system, canales push.

Codex CLI Harness

Tipo: App-Server propietario

# App-Server mode
codex app-server --listen stdio://

# El app-server:
# 1. Escucha JSON-RPC custom (no ACP puro)
# 2. Gestiona Threads, Turns, Items
# 3. Streama eventos al cliente

Ventaja: Primitivas muy claras (Thread/Turn/Item), VS Code extension oficial.

OpenCode Harness

Tipo: MCP + ACP nativos

# MCP (configurado)
~/.config/opencode/opencode.jsonc

# ACP (si disponible)
opencode --acp

# El agente:
# 1. Carga providers de ~/.local/share/opencode/auth.json
# 2. Ejecuta modelo seleccionado
# 3. Ejecuta herramientas (file edit, bash, grep, etc.)

Ventaja: Multi-modelo, integración directa con Copilot.
