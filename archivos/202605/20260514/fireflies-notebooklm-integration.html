Iteraciones de validación: 3 fuentes contrastadas por tema

📋 Resumen Ejecutivo

Esta investigación contrasta las posibilidades de integración entre Fireflies.ai y Google NotebookLM mediante Model Context Protocol (MCP) y APIs oficiales/comunitarias.

1️⃣ MCP para Fireflies.ai

✅ MCP OFICIAL DISPONIBLE

Fireflies.ai lanzó su MCP Server oficial en beta en marzo de 2026.

Detalles clave:

Aspecto Información
URL del servidor MCP https://api.fireflies.ai/mcp
Estado Beta (disponible públicamente)
Autenticación OAuth (Google/Microsoft) o API Key
Documentación oficial https://docs.fireflies.ai/
Blog de anuncio https://fireflies.ai/blog/fireflies-mcp-server/

Herramientas disponibles vía MCP:

  • get_user → Retorna información del perfil del usuario
  • get_transcript → Obtiene transcripción completa usando transcriptId
  • get_transcripts → Búsqueda en transcripciones pasadas con filtros

Configuración para Claude Desktop:

{
  "mcpServers": {
    "fireflies": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "https://api.fireflies.ai/mcp",
        "--header",
        "Authorization: Bearer YOUR_API_KEY_HERE"
      ]
    }
  }
}

Fuentes verificadas: 1. https://fireflies.ai/blog/fireflies-mcp-server/ (blog oficial) 2. https://guide.fireflies.ai/articles/8272956938-learn-about-the-fireflies-mcp-server-model-context-protocol (documentación oficial) 3. https://docs.fireflies.ai/getting-started/introduction (API docs)

2️⃣ API de Fireflies.ai

✅ API GraphQL DISPONIBLE

Características principales:

  • Tipo: GraphQL (no REST)
  • Endpoint único: Simplifica interacciones
  • Funcionalidades:
    • Upload de audio de reuniones
    • Fetch de transcripciones
    • Eliminación de transcripciones
    • Gestión de usuarios y roles
    • NLP personalizado para extracción de insights

Ventajas de GraphQL:

  1. Precisión: Retorna exactamente lo solicitado (sin over-fetching)
  2. Single endpoint: Más fácil de mantener
  3. Flexibilidad: Queries evolucionan con necesidades del proyecto

Documentación completa: https://docs.fireflies.ai/

Casos de uso documentados: - Canales de audio hacia Fireflies para transcripción - Extracción de keywords, sentences, transcripts completos - Gestión de datos de reuniones (eliminar, gestionar) - Información de usuarios y roles del equipo

3️⃣ Google NotebookLM - MCP e Integración

🟡 SITUACIÓN MIXTA

A) NotebookLM Enterprise (API Oficial)

✅ API OFICIAL DE GOOGLE CLOUD DISPONIBLE

Para usuarios empresariales de NotebookLM Enterprise, existe API oficial:

Endpoint base:

https://ENDPOINT_LOCATION-discoveryengine.googleapis.com/v1alpha/projects/PROJECT_NUMBER/locations/LOCATION/notebooks

Operaciones soportadas:

Método Operación Descripción
POST notebooks.create Crear notebook nuevo
GET notebooks.get Recuperar notebook específico
GET notebooks.listRecentlyViewed Listar últimos 500 notebooks
POST notebooks.batchDelete Eliminar notebooks
POST notebooks.share Compartir notebook con usuarios
POST notebooks.addSource Agregar fuentes (URL, texto, Drive, YouTube)

Requisitos: - Google Cloud project number - Licencias de NotebookLM Enterprise - Autenticación vía gcloud auth print-access-token

Fuentes oficiales: 1. https://docs.cloud.google.com/gemini/enterprise/notebooklm-enterprise/docs/api-notebooks 2. https://docs.cloud.google.com/gemini/enterprise/notebooklm-enterprise/docs/api-notebooks-sources

B) NotebookLM Consumer (Versión Gratuita)

❌ NO HAY API OFICIAL

Google no proporciona API pública para la versión consumer de NotebookLM.

✅ SOLUCIONES COMUNITARIAS (MCP Servers)

Existen múltiples MCP servers comunitarios para NotebookLM:

Repositorio Características Estado
PleasePrompto/notebooklm-mcp Chrome automation vía Patchright, stdio + HTTP, 32+ tools, multi-account ⭐ Más completo (v2.0.0)
m4yk3ldev/notebooklm-mcp 32 tools, CDP-based auth, Studio content generation Activo
alfredang/notebooklm-mcp Bridge para Claude Desktop/Code, generación de podcasts/videos/slides Activo
roomi-fields/notebooklm-mcp HTTP REST API + MCP, 33 endpoints para n8n/Zapier/Make Activo
Pantheon-Security/notebooklm-mcp-secure 17 capas de hardening de seguridad Reciente (3 días)
jacob-bd/notebooklm-mcp-cli CLI + MCP server Muy reciente (2 días)

Enlace principal: https://github.com/PleasePrompto/notebooklm-mcp

Herramientas comunes en MCP servers comunitarios:

  • ask_question → Preguntar sobre notebook con citas
  • add_source → Agregar URLs, texto, YouTube
  • generate_audio → Crear Audio Overview (podcast)
  • notebook_create/list/get/delete → Gestión de notebooks
  • download_audio → Descargar audio generado
  • research_start/status/import → Research autónomo (algunos)

Método de autenticación: - Login vía Chrome visible (una vez) - Cookies persistidas en perfil local - Sin almacenamiento encriptado de credenciales

Instalación típica:

npx notebooklm-mcp@latest

Configuración para Claude:

{
  "mcpServers": {
    "notebooklm": {
      "command": "npx",
      "args": ["notebooklm-mcp@latest"]
    }
  }
}

4️⃣ Estrategia de Integración Recomendada

Flujo: OpenClaw → Fireflies → NotebookLM

┌─────────────┐     ┌──────────────┐     ┌───────────────┐
│  OpenClaw   │────▶│  Fireflies   │────▶│   NotebookLM  │
│  (KIPP)     │     │  (MCP/API)   │     │   (MCP/API)   │
└─────────────┘     └──────────────┘     └───────────────┘
      │                    │                     │
      │ 1. Obtener        │ 2. Transcripción    │ 3. Análisis
      │    transcripción  │    y metadatos      │    y síntesis
      ▼                    ▼                     ▼

Opción A: Usando MCP Servers (Recomendado para prototipos)

  1. Fireflies MCP: https://api.fireflies.ai/mcp (oficial)
  2. NotebookLM MCP: npx notebooklm-mcp@latest (comunitario)
  3. Orquestación: OpenClaw puede actuar como coordinador entre ambos

Ventajas: - Sin código complejo - Autenticación manejada - Fácil configuración en Claude/Cursor

Desventajas: - NotebookLM MCP es no oficial (puede romperse) - Requiere Chrome para NotebookLM

Opción B: Usando APIs Directas (Producción)

  1. Fireflies API: GraphQL en https://docs.fireflies.ai/
  2. NotebookLM Enterprise API: Google Cloud (solo enterprise)
  3. Para consumer: Web scraping controlado o MCP comunitario

Ventajas: - Más estable y controlado - Sin dependencias de Chrome

Desventajas: - NotebookLM consumer sin API oficial - Requiere más desarrollo

5️⃣ Consideraciones de Seguridad

Fireflies MCP:

  • ✅ Usa API key existente o OAuth
  • ✅ TLS encryption
  • ⚠️ Zero-Day Retention NO aplica a datos procesados vía conectores externos
  • ⚠️ Datos en ChatGPT/Claude siguen políticas de OpenAI/Anthropic

NotebookLM MCP (comunitario):

  • ⚠️ No oficial - sin soporte de Google
  • ⚠️ Credenciales en perfil de Chrome local (sin encriptación)
  • ⚠️ Interfaz con endpoints internos (puede cambiar)
  • ✅ Código open-source (auditable)

Recomendación:

  • Usar solo para datos no sensibles
  • Revisar políticas de retención de cada plataforma
  • Considerar NotebookLM Enterprise para uso corporativo

7️⃣ Conclusiones

✅ Hallazgos Confirmados:

  1. Fireflies.ai TIENE MCP oficial (beta desde marzo 2026)
  2. Fireflies API es GraphQL con documentación completa
  3. NotebookLM Enterprise TIENE API oficial (Google Cloud)
  4. NotebookLM consumer NO tiene API oficial, pero existen 6+ MCP servers comunitarios activos
  5. Integración es viable usando combinación de MCP servers
