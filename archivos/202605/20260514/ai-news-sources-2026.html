4. MCP Servers para X/Twitter y Reddit

Composio MCP Integration

Twitter/X MCP

  • URL: https://composio.dev/toolkits/twitter
  • Proveedor: Composio
  • Tipo: MCP server con OAuth gestionado
  • Funciones principales:
    • Create posts (tweets)
    • Get bookmarks, followers, following
    • Manage lists, DMs
    • Media upload
    • Compliance jobs
  • Autenticación: OAuth a través de Composio Connect
  • Requisitos: API key de Composio + OAuth de Twitter

Reddit MCP

  • URL: https://composio.dev/toolkits/reddit
  • Proveedor: Composio
  • Tipo: MCP server con OAuth gestionado
  • Funciones principales:
    • Create/edit/delete posts y comentarios
    • Get listings (hot, new, top, controversial)
    • Search subreddits y posts
    • User information y preferencias
    • Subreddit rules y flairs
  • Autenticación: OAuth a través de Composio Connect
  • Requisitos: API key de Composio + OAuth de Reddit

XActions (Open Source)

  • URL: https://github.com/nirholas/XActions
  • Tipo: MCP server open-source para X/Twitter
  • Características:
    • Sin costos de API (usa scraping/browser automation)
    • CLI, Node.js library, MCP server, browser scripts
    • Auto follow/unfollow, likes, comments
    • Real-time analytics monitoring
    • No requiere API keys oficiales de Twitter
  • Licencia: Open source
  • Requisitos: Node.js, configuración local

Awesome MCP Servers

  • URL: https://github.com/punkpeye/awesome-mcp-servers
  • Tipo: Lista curada de MCP servers
  • Contenido: Social media MCPs y otros
  • Actualización: Comunitaria

5. Requisitos para Acceder a X/Twitter y Reddit

X/Twitter API

Opción Oficial (Twitter API v2)

  • Portal: https://developer.twitter.com/
  • Niveles de acceso:
    • Free: 1,500 tweets/mes (read-only), sin write access
    • Basic ($100/mes): 10,000 tweets/mes, write access limitado
    • Pro ($5,000/mes): 1M tweets/mes, full access
    • Enterprise: Custom pricing
  • Requisitos:
    • Cuenta de Twitter verificada (phone)
    • Developer account approval
    • App registration con use case description
    • API keys (Bearer token, API key/secret)
  • Documentación 2026: https://elfsight.com/blog/how-to-get-x-twitter-api-key-in-2026/

Opción Alternativa (XActions)

  • Sin API oficial: Usa browser automation/scraping
  • Requisitos:
    • Cuenta de Twitter activa
    • Node.js instalado
    • Variables de entorno para credenciales (username/password o cookies)
  • Ventajas: Sin costos, sin rate limits oficiales
  • Riesgos: Posible violación de ToS de Twitter, menos estable

Reddit API

Opción Oficial (Reddit API)

  • Portal: https://www.reddit.com/dev/api/
  • Autenticación: OAuth 2.0
  • Requisitos:
    • Cuenta de Reddit
    • App registration en https://www.reddit.com/prefs/apps
    • Client ID y Client Secret
    • Redirect URI configurado
  • Rate limits:
    • 60 requests/minuto (OAuth)
    • Varía por endpoint
  • Documentación: https://www.reddit.com/dev/api/

Opción Composio MCP

  • URL: https://composio.dev/toolkits/reddit/framework/opencode
  • Ventajas:
    • OAuth gestionado automáticamente
    • Simplifica integración con agentes AI
    • Maneja token refresh
  • Requisitos:
    • API key de Composio
    • Autorizar Reddit a través de Composio Connect
  • Documentación: https://composio.dev/toolkits/reddit/framework/opencode
