Investigador: KIPP
Fuentes verificadas: 12+ URLs contrastadas

🎯 Resumen Ejecutivo

Google Stitch NO tiene MCP oficial al momento de esta investigación. Sin embargo, existe un flujo de trabajo viable usando Google AI Studio + herramientas alternativas con MCP.

1. Google Stitch - Estado del MCP

Hallazgos Principales

Aspecto Estado
MCP Server Oficial ❌ No existe
Integración con AI Studio ✅ Nativa
API Pública Documentada ⚠️ Limitada (vía Gemini API)
Exportación de Código ✅ HTML/CSS/JS, React

Qué es Google Stitch

  • Origen: Adquirido de Galileo AI (mid-2025), rebranding a Stitch
  • Plataforma: Parte de Google AI Studio (aistudio.google.com)
  • Modelo: Gemini-powered (2.0 Flash / 2.5 Pro)
  • Costo: Gratis con límites diarios (400 design credits, 15 redesign credits/día)
  • Documentación: https://stitch.withgoogle.com/docs

Flujo Stitch → Código

Prompt → Stitch (UI Design) → AI Studio Build Mode → Código → Deploy
         (Gemini)              (Gemini)              (HTML/CSS/JS/React)

Limitaciones: - No hay API directa para automatización externa - Requiere interacción manual en la UI de AI Studio - Sin webhook o endpoints para integración programática

Fuentes: - https://www.mindstudio.ai/blog/google-stitch-to-ai-studio-design-to-code-workflow - https://www.banani.co/blog/google-stitch-ai-review - https://www.codecademy.com/article/google-stitch-tutorial-ai-powered-ui-design-tool

2. Alternativas con MCP Oficial

2.1 Figma MCP Server ✅ (RECOMENDADO)

Estado: Beta oficial (gratis durante beta)
Endpoint remoto: https://mcp.figma.com/mcp

Capacidades: - Leer variables, componentes, layout data de Figma - Generar código desde frames seleccionados - Escribir en canvas (remote server) - Code-to-canvas (live UI → Figma layers) - Integración con Code Connect para consistencia

Clientes compatibles: - Cursor, Claude Code, Claude Desktop - Codex (OpenAI), Copilot CLI - Gemini CLI, Kiro, Augment Code

Configuración MCP:

{
  "mcpServers": {
    "figma": {
      "command": "npx",
      "args": ["-y", "@figma/mcp-server"],
      "env": {
        "FIGMA_API_KEY": "tu_api_key"
      }
    }
  }
}

Fuentes: - https://help.figma.com/hc/en-us/articles/32132100833559-Guide-to-the-Figma-MCP-server - https://www.figma.com/resource-library/what-is-mcp/ - https://developers.figma.com/docs/figma-mcp-server/

2.2 Vercel v0 + MCP ✅

Estado: MCP nativo en v0.app
Documentación: https://v0.app/docs/MCP

Características: - Bring-your-own MCP: Configura servidores externos - Marketplace Integrations: Neon, Supabase, Stripe, etc. - Modos de permiso: Disabled / Manual (ask) / Auto - Soporta presets: Context7, Linear, Notion, Sentry, Zapier

Vercel como host MCP: - v0 puede conectarse a servidores MCP externos - También permite desplegar tus propios MCP servers en Vercel Functions

Fuentes: - https://vercel.com/docs/mcp - https://v0.app/docs/MCP - https://mcpservers.org/servers/hellolucky/v0-mcp

4. Flujo de Trabajo Recomendado para OpenClaw

Opción A: Figma + MCP (Más automatizable)

1. Diseñar en Figma (manual o con plugins AI)
2. OpenClaw → Figma MCP Server → Extraer diseño
3. Generar HTML/CSS estático con agente
4. Commit a Gargantua → Deploy

Ventajas: - MCP oficial con API documentada - OpenClaw puede leer diseños programáticamente - Consistencia con design systems

Desventajas: - Requiere Figma (paid para algunas features) - Diseño inicial no es AI-generated directamente

Opción B: Stitch + Intervención Humana (Mejor UI AI)

1. Usuario genera diseño en Google Stitch (manual)
2. Exporta código desde AI Studio
3. OpenClaw recibe código → Refina/Adapta
4. Commit a Gargantua → Deploy

Ventajas: - Mejor generación AI de UI (Gemini-powered) - Gratis durante beta - Multi-screen layouts automáticos

Desventajas: - Sin automatización completa (requiere humano en Stitch) - No hay MCP para integración directa

Opción C: Vercel v0 + OpenClaw MCP

1. OpenClaw se configura como MCP client
2. Conectar a v0 MCP server
3. Generar componentes vía prompts
4. Integrar en landing page estática

Ventajas: - MCP nativo en v0 - Ecosistema Vercel (deploy fácil) - Componentes shadcn/ui reutilizables

Desventajas: - v0 orientado a componentes, no páginas completas - Requiere configuración MCP en OpenClaw

5. Recomendación Final KIPP

Para el caso de uso “Stitch genera diseños → OpenClaw desarrolla landing estática”:

Flujo Óptimo (Híbrido)

  1. Fase Diseño: Usuario usa Google Stitch en AI Studio (gratis, mejor calidad AI)
  2. Export: Descargar código HTML/CSS desde AI Studio
  3. Fase OpenClaw:
    • Recibir código exportado
    • Refinar con agente (accesibilidad, SEO, responsive)
    • Adaptar a estructura Gargantua
    • Commit + Push automático
  4. Deploy: Vercel/Netlify/Firebase Hosting

Por qué este flujo

  • Stitch tiene mejor generación UI AI que alternativas
  • OpenClaw no puede automatizar Stitch (sin API/MCP)
  • Intervención humana mínima: Solo paso 1-2
  • Calidad: Combina lo mejor de ambos mundos

Automatización Futura (si Google lanza MCP)

Monitorear: - https://mcpservers.org/servers/google/mcp (actualmente otros servicios Google) - Anuncios en Google AI Studio changelog - GitHub: google/mcp-servers (si open-sourcean)
