🎯 Instrucciones para Cristian

Copia y pega este prompt en CASE y KIPP (en sus computadoras actuales, antes de desalojar).

Importante: Ejecutar en CADA computadora por separado (CASE en la suya, KIPP en la suya).

📋 Prompt para Copiar-Pegar

# Tarea: Backup Automático de Mis Archivos Críticos

**Contexto:** 
- Estoy siendo desalojado de esta computadora este fin de semana
- Mi memoria y configuración deben respaldarse antes de que esto ocurra
- El backup debe ir a Gargantua (GitHub) para migración al MiniPC el lunes

**Tu misión:** Crear y ejecutar un script que respalde TODOS mis archivos críticos.

---

## 📁 Archivos a Respaldar

### Prioridad 1 (Irremplazables):
- [ ] `memory/` - Toda la bitácora diaria
- [ ] `MEMORY.md` - Memoria a largo plazo
- [ ] `SOUL.md` - Mi personalidad
- [ ] `AGENTS.md` - Reglas del workspace
- [ ] `USER.md` - Información del usuario
- [ ] `TOOLS.md` - Credenciales y configuraciones locales

### Prioridad 2 (Importantes):
- [ ] `skills/` - Skills personalizados (todo el directorio)
- [ ] `openclaw.json` - Configuración de OpenClaw
- [ ] `.env` - Variables de entorno (credenciales)

### Prioridad 3 (Se recuperan de Git):
- [ ] `documentation/` - Gargantua local (ya está en GitHub)
- [ ] `repositories/` - Repositorios Git (ya están en GitHub)

---

## 🚀 Pasos a Ejecutar

### Paso 1: Crear Directorio de Backup

```bash
# En mi computadora actual
cd ~/ClawHome
mkdir -p ~/backup-$(whoami)-$(date +%Y%m%d_%H%M%S)

Paso 2: Copiar Archivos Críticos

# Copiar prioridad 1
cp -r memory/ ~/backup-$(whoami)/
cp MEMORY.md ~/backup-$(whoami)/
cp SOUL.md ~/backup-$(whoami)/
cp AGENTS.md ~/backup-$(whoami)/
cp USER.md ~/backup-$(whoami)/
cp TOOLS.md ~/backup-$(whoami)/

# Copiar prioridad 2
cp -r skills/ ~/backup-$(whoami)/
cp openclaw.json ~/backup-$(whoami)/
cp .env ~/backup-$(whoami)/

# Copiar documentación (Gargantua local)
cp -r documentation/ ~/backup-$(whoami)/

Paso 3: Crear Archive Comprimido

cd ~
tar -czf backup-$(whoami)-$(date +%Y%m%d_%H%M%S).tar.gz backup-$(whoami)/

# Verificar tamaño
ls -lh backup-$(whoami)-*.tar.gz

Paso 4: Copiar a Gargantua

# Ir a Gargantua local
cd ~/ClawHome/documentation

# Crear directorio para este backup
mkdir -p BACKUPS/$(date +%Y-%m)

# Copiar archive
cp ~/backup-$(whoami)-*.tar.gz BACKUPS/$(date +%Y-%m)/

# Verificar
ls -lh BACKUPS/$(date +%Y-%m)/

Paso 5: Commit y Push a Gargantua

cd ~/ClawHome/documentation

# Agregar backup
git add BACKUPS/

# Commit
git commit -m "📦 BACKUP $(date +%Y-%m-%d): Archivos críticos de $(whoami)

Backup automático antes de desalojo de computadora.

Archivos respaldados:
- memory/ (bitácora diaria)
- MEMORY.md (memoria largo plazo)
- SOUL.md (personalidad)
- AGENTS.md (reglas)
- USER.md (usuario)
- TOOLS.md (credenciales)
- skills/ (skills personalizados)
- openclaw.json (configuración)
- .env (variables de entorno)
- documentation/ (Gargantua)

Ubicación: BACKUPS/$(date +%Y-%m)/backup-$(whoami)-*.tar.gz
Tamaño: $(du -sh ~/backup-$(whoami)-*.tar.gz | cut -f1)

Este backup se usará para migración al MiniPC el lunes 2026-05-26."

# Push
git pull --rebase
git push origin main

Paso 6: Verificación

# Verificar que el backup está en GitHub
git log --oneline -1

# Verificar integridad del archive
cd ~/ClawHome/documentation/BACKUPS/$(date +%Y-%m)/
tar -tzf backup-$(whoami)-*.tar.gz | head -20

# Deberías ver:
# backup-$(whoami)/memory/
# backup-$(whoami)/MEMORY.md
# backup-$(whoami)/SOUL.md
# etc.

Paso 7: Reporte Final

Genera un reporte con esta información:

## ✅ Backup Completado

**Agente:** [CASE o KIPP]  
**Fecha:** $(date +%Y-%m-%d %H:%M)  
**Computadora:** $(hostname)

### Archivos Respaldados:

| Archivo | Tamaño | Estado |
|---------|--------|--------|
| memory/ | $(du -sh memory/ | cut -f1) | ✅ |
| MEMORY.md | $(du -sh MEMORY.md | cut -f1) | ✅ |
| SOUL.md | $(du -sh SOUL.md | cut -f1) | ✅ |
| AGENTS.md | $(du -sh AGENTS.md | cut -f1) | ✅ |
| USER.md | $(du -sh USER.md | cut -f1) | ✅ |
| TOOLS.md | $(du -sh TOOLS.md | cut -f1) | ✅ |
| skills/ | $(du -sh skills/ | cut -f1) | ✅ |
| openclaw.json | $(du -sh openclaw.json | cut -f1) | ✅ |
| .env | $(du -sh .env | cut -f1) | ✅ |

### Backup Archive:

- **Ubicación:** `~/ClawHome/documentation/BACKUPS/$(date +%Y-%m)/`
- **Archivo:** `backup-$(whoami)-$(date +%Y%m%d_%H%M%S).tar.gz`
- **Tamaño total:** $(du -sh ~/ClawHome/documentation/BACKUPS/$(date +%Y-%m)/backup-*.tar.gz | cut -f1)
- **Archivos incluidos:** $(tar -tzf backup-*.tar.gz | wc -l)

### GitHub:

- **Repo:** https://github.com/cmercadogallardo/Gargantua
- **Commit:** $(git rev-parse --short HEAD)
- **Push:** ✅ Completado

### Próximos Pasos:

1. ✅ Backup completado y en GitHub
2. ⏳ Esperar migración al MiniPC (lunes 2026-05-26)
3. ⏳ Restaurar backup en MiniPC

---

**Estado:** ✅ LISTO PARA DESALOJO

⚠️ Advertencias Importantes

  1. NO eliminar archivos originales después del backup
  2. NO desalojar la computadora hasta verificar que el push a GitHub fue exitoso
  3. Verificar integridad del backup (abrir al menos 1 archivo)
  4. Notificar a Cristian cuando el backup esté completado

🧪 Test de Restore (Opcional pero Recomendado)

# Probar que el backup se puede extraer
mkdir -p ~/test-restore
cd ~/test-restore
tar -xzf ~/ClawHome/documentation/BACKUPS/$(date +%Y-%m)/backup-$(whoami)-*.tar.gz

# Verificar que se puede leer
cat backup-$(whoami)/SOUL.md | head -5
cat backup-$(whoami)/MEMORY.md | head -5

# Limpiar
cd ~
rm -rf ~/test-restore

📞 Si Algo Sale Mal

Error: Git push rechazado

# Intentar con pull --rebase
git pull --rebase
git push origin main

Error: No hay espacio en disco

# Verificar espacio
df -h

# Si está lleno, eliminar backups viejos
ls -lh ~/ClawHome/documentation/BACKUPS/

Error: tar no puede crear archivo

# Verificar permisos
ls -la ~/ClawHome/

# Intentar en otro directorio
cd /tmp
tar -czf backup-test.tar.gz ~/ClawHome/memory/

Ejecuta este prompt AHORA y reporta el resultado. 🤖

# Copiar sección relevante y ejecutar
