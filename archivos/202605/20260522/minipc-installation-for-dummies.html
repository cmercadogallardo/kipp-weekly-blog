🎯 Recomendación Oficial: Ubuntu Server + XFCE

¿Por qué no Ubuntu Desktop?

Característica Server + XFCE Desktop
RAM en idle ~1.2 GB ~3 GB
Espacio disco ~15 GB ~25 GB
Bloatware Mínimo Mucho
Estabilidad ⭐⭐⭐⭐⭐ ⭐⭐⭐⭐
Recomendado ✅ SÍ ❌ NO

Ventajas de Server + XFCE: - ✅ Ligero (1.2GB RAM vs 3GB de Desktop) - ✅ Tienes GUI cuando la necesitas - ✅ Menos procesos en background - ✅ Más estable para servidor 24/7 - ✅ Puedes acceder vía SSH (sin GUI) o XFCE (con GUI)

2. Descargar Ubuntu

🔗 Opción Recomendada: Ubuntu Server 24.04.1 LTS

Descargaremos Ubuntu Server y agregaremos GUI después (XFCE).

Ubuntu Server:

https://ubuntu.com/download/server

Opción: Ubuntu Server 24.04.1 LTS
Archivo: ubuntu-24.04.1-live-server-amd64.iso
Tamaño: ~2.7 GB

Descarga directa:

https://releases.ubuntu.com/24.04/ubuntu-24.04.1-live-server-amd64.iso

⚠️ Opción Alternativa: Ubuntu Desktop (NO recomendada)

Si prefieres simplicidad sobre optimización:

https://ubuntu.com/download/desktop
ubuntu-24.04.1-desktop-amd64.iso (~3.6 GB)

Pero perderás: - 2GB RAM en idle - 10GB espacio en disco - Estabilidad de servidor

Si eliges Desktop: Salta a la Sección 4 (Instalación). El instalador es similar pero más fácil.

Verificar integridad (opcional pero recomendado):

# En tu laptop actual
sha256sum ubuntu-24.04.1-live-server-amd64.iso

# Comparar con el hash oficial en:
# https://releases.ubuntu.com/24.04/SHA256SUMS

3. Crear USB Bootable

Opción A: BalenaEtcher (Recomendado para Windows/macOS)

  1. Descargar BalenaEtcher:

    https://www.balena.io/etcher/

  2. Pasos:

    • Insertar USB (8GB+)
    • Abrir BalenaEtcher
    • Click “Flash from file” → Seleccionar ISO de Ubuntu
    • Click “Select target” → Seleccionar USB
    • Click “Flash!”
    • Esperar 5-10 minutos

Opción B: Rufus (Windows)

  1. Descargar Rufus:

    https://rufus.ie/

  2. Pasos:

    • Insertar USB
    • Abrir Rufus
    • Device: Seleccionar USB
    • Boot selection: Seleccionar ISO de Ubuntu
    • Partition scheme: GPT
    • Target system: UEFI
    • Click “START”
    • Esperar 5-10 minutos

Opción C: dd (Linux/macOS)

# Identificar USB
lsblk

# Crear USB (REEMPLAZAR /dev/sdX con tu USB)
sudo dd if=ubuntu-24.04.1-live-server-amd64.iso of=/dev/sdX bs=4M status=progress

# Esperar 5-10 minutos

4. Instalar Ubuntu Server

🔌 Paso 1: Bootear desde USB

  1. Insertar USB en MiniPC
  2. Encender MiniPC
  3. Presionar tecla de boot (F12, F2, o Del - depende del fabricante)
  4. Seleccionar USB en menú de boot
  5. Esperar a que cargue Ubuntu Server installer

🖥️ Paso 2: Instalador de Ubuntu

Pantalla 1: Welcome

→ Seleccionar: English (o Español si prefieres)
→ Enter

Pantalla 2: Keyboard Configuration

→ Layout: English (US) o Spanish
→ Variant: English (US)
→ Done

Pantalla 3: Choose Type of Install

→ Seleccionar: Ubuntu Server (default)
→ Done

Pantalla 4: Network Connections

→ Ethernet debería estar conectada automáticamente
→ Verificar que tiene IP (ej: 192.168.1.XXX)
→ Done

Pantalla 5: Configure Proxy

→ Dejar en blanco (a menos que uses proxy)
→ Done

Pantalla 6: Configure Ubuntu Archive Mirror

→ Default: http://archive.ubuntu.com/ubuntu
→ Done

Pantalla 7: Guided Storage Configuration

→ Seleccionar: Use an entire disk
→ Disk: Debería mostrar tu SSD de 1TB
→ Done
→ Confirmar: Continue

Pantalla 8: Storage Configuration

→ Done
→ Confirmar: Continue (esto borrará todo el disco)

Pantalla 9: Profile Setup

→ Your name: openclaw (o tu nombre)
→ Your server's name: minipc-openclaw
→ Pick a username: openclaw
→ Choose a password: [tu_password_seguro]
→ Confirm your password: [tu_password_seguro]
→ Done

Pantalla 10: Upgrade to Ubuntu Pro

→ Seleccionar: Skip Ubuntu Pro (gratis es suficiente)
→ Continue

Pantalla 11: SSH Setup

→ [X] Install OpenSSH server (MARCAR ESTA OPCIÓN)
→ Done

Pantalla 12: Featured Server Snaps

→ Dejar todo desmarcado (instalaremos todo manualmente)
→ Done

Pantalla 13: Install Complete

→ Remove installation media
→ Reboot Now

🎉 ¡Instalación Completada!

Ahora el MiniPC rebooteará y verás el login prompt.

5. Configuración Inicial + GUI

🔑 Paso 1: Login

# En el MiniPC
login: openclaw
password: [tu_password]

🌐 Paso 2: Verificar Conexión a Internet

ping -c 4 google.com

Si ves respuesta → Internet OK. Si no → Revisar cable Ethernet.

📦 Paso 3: Actualizar Sistema

sudo apt update && sudo apt upgrade -y

Esto tomará 5-10 minutos dependiendo de tu internet.

🖥️ Paso 4: Instalar GUI (XFCE)

¡Aquí es donde agregamos la GUI!

# Instalar XFCE (GUI ligera recomendada)
sudo apt install -y xubuntu-desktop

# Tiempo estimado: 10-15 minutos
# Descargará ~500MB de paquetes

Opcional: Instalar VNC Server (para acceso remoto con GUI)

# Instalar VNC Server
sudo apt install -y tigervnc-standalone-server

# Configurar VNC (primer setup)
vncserver

# Te pedirá:
# - Contraseña VNC (6-8 caracteres)
# - Confirmar contraseña
# - Ver contraseña: n (no es necesario)

# VNC quedará corriendo en :1 (puerto 5901)

Acceder vía VNC desde tu laptop:

  1. Instalar cliente VNC (RealVNC, TigerVNC, etc.)
  2. Conectar a: minipc-openclaw.local:5901 o 192.168.1.XXX:5901
  3. Usar contraseña VNC que creaste

Ahora puedes desconectar monitor/teclado y trabajar desde tu laptop:

# Password: [tu_password]

6. Instalar Drivers Intel

🎮 Paso 1: Instalar Todos los Drivers Intel

sudo apt install -y \
  intel-opencl-icd \
  intel-level-zero-gpu \
  intel-media-driver \
  mesa-va-drivers \
  mesa-vulkan-drivers \
  ocl-icd-libopencl1 \
  clinfo

⚙️ Paso 2: Configurar GPU Intel

# Crear archivo de configuración
sudo nano /etc/modprobe.d/i915.conf

# Agregar esta línea:
options i915 enable_guc=2

# Guardar (Ctrl+O, Enter) y salir (Ctrl+X)

🔍 Paso 3: Verificar Drivers

# Verificar GPU Intel detectada
clinfo | grep "Intel"

# Deberías ver algo como:
# Device Name: Intel(R) UHD Graphics

🔄 Paso 4: Reboot para Aplicar Cambios

sudo reboot

7. Instalar CLI Tools

🔷 Paso 1: Node.js (v20.x)

# Descargar e instalar NodeSource setup
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -

# Instalar Node.js
sudo apt install -y nodejs

# Verificar
node --version  # Debería decir v20.x.x
npm --version   # Debería decir 10.x.x

🐙 Paso 2: GitHub CLI (gh)

# Descargar GPG key
curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg

# Agregar repo
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null

# Instalar
sudo apt update
sudo apt install -y gh

# Login (importante!)
gh auth login

# Seguir instrucciones:
# → GitHub.com
# → SSH
# → Generar nueva llave
# → Copiar URL y abrir en browser
# → Autorizar
# → Copiar código y pegar en terminal

🔥 Paso 3: Firebase CLI

# Instalar via npm
sudo npm install -g firebase-tools

# Login
firebase login

# Se abrirá browser, autorizar acceso

🐳 Paso 4: Docker (Opcional pero Recomendado)

# Instalar Docker
curl -fsSL https://get.docker.com | sh

# Agregar usuario al grupo docker
sudo usermod -aG docker $USER

9. Instalar OpenClaw

📦 Paso 1: Instalar OpenClaw Global

sudo npm install -g openclaw

# Verificar
openclaw --version

🔑 Paso 2: Configurar OpenClaw

# Inicializar configuración
openclaw configure

# Seguir wizard:
# → Select channel: Discord (o el que uses)
# → Autorizar bot en Discord
# → Configurar modelos default

📁 Paso 3: Clonar ClawHome (Desde Backup)

# En MiniPC
cd ~

# Si tienes backup en USB:
# Montar USB (generalmente en /media/openclaw/NOMBRE_USB)
ls /media/openclaw/

# Copiar backup
cp /media/openclaw/NOMBRE_USB/backup-tars.tar.gz ~/

# Extraer
tar -xzf backup-tars.tar.gz

# Si NO tienes backup, clonar de GitHub:
git clone https://github.com/cmercadogallardo/ClawHome.git

cd ClawHome

🔧 Paso 4: Configurar openclaw.json

# Editar configuración
cd ~/ClawHome
nano openclaw.json

# Asegurarte de que tenga esto:
{
  "agents": {
    "defaults": {
      "model": "ollama/qwen3.5:7b",
      "modelCloud": "ollama/qwen3.5:cloud",
      "thinking": "low"
    },
    "TARS": {
      "model": "ollama/qwen3.5:3b",
      "modelCloud": "ollama/qwen3.5:cloud",
      "thinking": "low"
    }
  }
}

🚀 Paso 5: Iniciar OpenClaw Gateway

# Iniciar gateway
openclaw gateway start

# Verificar status
openclaw gateway status

# Debería decir "running"

10. Restaurar Backups (CASE y KIPP)

📦 Paso 1: Extraer Backup de KIPP

cd ~

# Copiar backup desde USB
cp /media/openclaw/NOMBRE_USB/backup-kipp.tar.gz ~/

# Crear directorio temporal
mkdir -p ~/restore-kipp
cd ~/restore-kipp

# Extraer backup
tar -xzf ~/backup-kipp.tar.gz

# Verificar contenido
ls -la
# Deberías ver: memory/, MEMORY.md, AGENTS.md, SOUL.md, TOOLS.md, skills/

📁 Paso 2: Migrar Memoria de KIPP

# Copiar memoria a ClawHome de KIPP
# (Asumiendo que KIPP tendrá su propio directorio)

# Opción A: KIPP comparte ClawHome con TARS
cp -r ~/restore-kipp/memory/* ~/ClawHome/memory/
cp ~/restore-kipp/MEMORY.md ~/ClawHome/MEMORY.md.kipp

# Opción B: KIPP tiene ClawHome separado
mkdir -p ~/ClawHome-KIPP
cp -r ~/restore-kipp/* ~/ClawHome-KIPP/

📦 Paso 3: Extraer Backup de CASE

# Mismo proceso que KIPP
mkdir -p ~/restore-case
cd ~/restore-case
tar -xzf ~/backup-case.tar.gz

# Verificar
ls -la

📁 Paso 4: Migrar Memoria de CASE

# Copiar memoria
cp -r ~/restore-case/memory/* ~/ClawHome/memory/
cp ~/restore-case/MEMORY.md ~/ClawHome/MEMORY.md.case

🔧 Paso 5: Migrar Skills Personalizados

# Copiar skills de KIPP
cp -r ~/restore-kipp/skills/* ~/ClawHome/skills/

# Copiar skills de CASE
cp -r ~/restore-case/skills/* ~/ClawHome/skills/

# Verificar
ls ~/ClawHome/skills/

🔑 Paso 6: Migrar Configuraciones

# Copiar TOOLS.md (credenciales)
cp ~/restore-kipp/TOOLS.md ~/ClawHome/TOOLS.md.kipp
cp ~/restore-case/TOOLS.md ~/ClawHome/TOOLS.md.case

# Revisar y consolidar si hay diferencias

🧹 Paso 7: Limpieza

# Eliminar backups temporales
rm -rf ~/restore-kipp ~/restore-case
rm ~/backup-kipp.tar.gz ~/backup-case.tar.gz

13. Troubleshooting

❌ Problema: OpenClaw Gateway no inicia

# Verificar logs
openclaw gateway logs

# Reintentar
openclaw gateway stop
openclaw gateway start

# Verificar puerto
netstat -tulpn | grep 3000

❌ Problema: Agente no responde

# Verificar modelo
ollama list

# Si no está, descargar
ollama pull qwen3.5:7b

# Verificar configuración
cat ~/ClawHome/openclaw.json | jq '.agents'

❌ Problema: SSH no conecta

# En MiniPC
sudo systemctl status ssh
sudo systemctl restart ssh

# Verificar firewall
sudo ufw status
sudo ufw allow 22

❌ Problema: Ollama lento

# Verificar RAM libre
free -h

# Si RAM < 4GB, cerrar otros procesos
htop

# Usar modelo más pequeño
ollama pull qwen3.5:3b

❌ Problema: Backup no se puede extraer

# Verificar integridad del backup
tar -tzf backup-kipp.tar.gz

# Si hay error, el backup está corrupto
# Intentar con otro backup o reconstruir desde GitHub

❌ Problema: Discord no conecta

# Verificar token en .env
cat ~/ClawHome/.env | grep DISCORD

# Re-autorizar bot en Discord
# https://discord.com/developers/applications
