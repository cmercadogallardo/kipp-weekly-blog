E - EXAMINE (Examinar)

1. Plugin github-copilot (Nativo OpenClaw)

Archivo clave: /opt/homebrew/lib/node_modules/openclaw/dist/extensions/github-copilot/index.js

Hallazgos:

// Auth method: device_code (OAuth nativo)
auth: [{
  id: "device",
  label: "GitHub device login",
  hint: "Browser device-code flow",
  kind: "device_code",
  run: async (ctx) => await runGitHubCopilotAuth(ctx)
}]

Variables de entorno soportadas: - COPILOT_GITHUB_TOKEN - GH_TOKEN - GITHUB_TOKEN

Token endpoint: https://api.github.com/copilot_internal/v2/token

Base URL por defecto: https://api.individual.githubcopilot.com

2. Plugin copilot-proxy (Alternativo)

Archivo clave: /opt/homebrew/lib/node_modules/openclaw/dist/extensions/copilot-proxy/index.js

Propósito: Conectar con proxy local de VS Code (puerto 3000)

Configuración:

DEFAULT_BASE_URL = "http://localhost:3000/v1"
DEFAULT_MODEL_IDS = [
  "gpt-5.2", "gpt-5.2-codex", "gpt-5.1",
  "claude-opus-4.6", "claude-sonnet-4.5",
  "gemini-3-pro", "grok-code-fast-1"
]

Nota: Requiere extensión VS Code “Copilot Proxy” corriendo localmente.

3. ACPX Runtime

Versión instalada: 0.5.3

Agent alias para Copilot:

copilot → copilot --acp --stdio

Comando de verificación:

acpx --version
# Output: 0.5.3

acpx copilot --help
# Disponible (sin output = disponible)

4. CLI copilot (GitHub oficial)

Versión: 1.0.48

Soporte ACP confirmado:

copilot --acp --stdio
# Inicia servidor ACP sobre stdio

Flags relevantes: - --acp: Modo Agent Client Protocol - --stdio: Comunicación vía stdin/stdout - --allow-all: Permisos automáticos (no-interactive)

5. OpenCode (Provider Separado)

Hallazgo crítico: OpenCode NO es necesario para GitHub Copilot.

OpenCode es un provider diferente que usa: - API keys (OPENCODE_API_KEY, OPENCODE_ZEN_API_KEY) - Catálogo Zen (multi-modelo: Claude, GPT, Gemini) - No usa OAuth de GitHub

Auth file verificado:

{
  "github-copilot": {
    "type": "oauth",
    "refresh": "[REDACTED]",
    "access": "[REDACTED]",
    "expires": 0
  }
}

✅ GitHub Copilot YA está autenticado en OpenCode (pero OpenClaw usa su propio flow nativo).

R - REPORT (Reportar)

Conclusiones Principales

  1. ✅ OAuth Nativo DISPONIBLE
    • OpenClaw tiene integración nativa con GitHub Copilot vía OAuth device flow
    • No requiere intermediarios (OpenCode, proxies, etc.)
    • Token caching y refresh automático incluidos
  2. ✅ ACP Runtime FUNCIONAL
    • acpx v0.5.3 instalado y operativo
    • Agent alias copilot configurado (copilot --acp --stdio)
    • Copilot CLI v1.0.48 soporta modo ACP nativamente
  3. ❌ OpenCode NO NECESARIO para Copilot
    • OpenCode es un provider separado (API key, catálogo Zen)
    • Usar OpenCode como “puente” a Copilot añade complejidad innecesaria
    • La skill opencode-github-copilot está desactualizada/mal enfocada
  4. ✅ Auth OAuth VIGENTE
    • Token GitHub Copilot presente en ~/.local/share/opencode/auth.json
    • Tipo: OAuth (gho_*)
    • Expiración: 0 (persistente/refreshable)

Recomendación Final

Quédate con OAuth nativo + ACPX. NO migres a OpenCode.

C - CONCLUDE (Concluir)

Veredicto Final

¿Vale la pena migrar a OpenCode como puente para GitHub Copilot?

❌ NO

Razones:

  1. Redundancia: OpenClaw ya tiene integración nativa OAuth con Copilot
  2. Complejidad: OpenCode añade otra capa de autenticación y gestión
  3. Latencia: Bridge añade overhead innecesario
  4. Mantenimiento: Dos configs que mantener vs una
  5. Feature parity: OAuth nativo tiene acceso completo a Copilot API
