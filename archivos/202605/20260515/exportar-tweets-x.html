Cómo Exportar TODOS los Tweets de una Cuenta de X (Twitter)

Fecha de investigación: 2026-05-15
Investigador: KIPP 🤖
Fuentes consultadas: help.x.com, Tweet Archivist, Circleboom, WikiHow, Reddit r/DataHoarder

📋 Resumen Ejecutivo

Existen dos métodos principales para exportar tweets de X (Twitter):

  1. Método oficial (GRATUITO) - Archive de datos personales desde Settings de X
    • ✅ Incluye TODOS los tweets desde la creación de la cuenta
    • ✅ Incluye tweets eliminados
    • ✅ Incluye likes, DMs, media, perfil completo
    • ⏱️ Tiempo de espera: 24-72 horas
    • 📦 Formato: ZIP con HTML + JSON/JS
  2. Herramientas de terceros - Circleboom, TweetDelete, etc.
    • ✅ Exportación inmediata
    • ✅ Formato CSV/Excel listo para analizar
    • ⚠️ Límite: ~3,200 tweets más recientes (API restriction)
    • 💰 Algunas son de pago

🎯 Método 1: Archive Oficial de X (Recomendado para backup completo)

¿Qué incluye el archivo de exportación?

Tipo de Dato ¿Incluido? Detalles
Todos los tweets ✅ Sí Originales, retweets, quote tweets, replies
Tweets eliminados ✅ Sí Se preservan aunque los hayas borrado
Likes ✅ Sí Historial completo de tweets que diste like
Direct Messages ✅ Sí Conversaciones individuales y grupales
Media (fotos/videos) ⚠️ Parcial Referencias y thumbnails, no siempre full-resolution
Perfil ✅ Sí Nombre, bio, fotos de perfil/header, historial de cambios
Followers/Following ✅ Sí Listas con user IDs (no siempre usernames completos)
Moments y Listas ✅ Sí Si creaste alguno
Datos de anuncios ✅ Sí Interacciones con promoted tweets
Configuración ✅ Sí Historial de personalización y privacidad

📁 Formato del archivo exportado

  • Contenedor: ZIP compressado

  • Tamaño: Desde pocos MB hasta varios GB (depende de media)

  • Estructura interna:

    your_archive.zip
├── Your_archive.html (visor tipo Twitter para navegar offline)
├── README.txt (instrucciones)
├── data/
│   ├── tweets.js (JSON wrapped en JS - TODOS tus tweets)
│   ├── account.js (información de cuenta)
│   ├── profile.js (detalles del perfil)
│   ├── direct-messages.js (todos los DMs)
│   ├── like.js (todos los likes)
│   ├── follower.js (lista de followers)
│   └── following.js (lista de following)
└── assets/ (CSS, JS, imágenes para el visor HTML)

Nota técnica: Los archivos .js en la carpeta data/ son JSON válido envuelto en una declaración JavaScript tipo:

window.YTD.tweets.part0 = [{...datos en JSON...}]

Para usarlos con parsers JSON, hay que quitar el wrapper y quedarse solo con el array.

🔧 Pasos detallados para solicitar la exportación

Desde Desktop (Navegador web)

  1. Ir a X.com e iniciar sesión en tu cuenta
  2. Click en “More” (tres puntos horizontales) en el panel izquierdo
  3. Seleccionar “Settings and privacy”
  4. Click en “Your account” en el menú
  5. Seleccionar “Download an archive of your data”
  6. Verificar identidad:
    • Ingresar contraseña de la cuenta
    • Ingresar código enviado a email o teléfono registrado
  7. Click en “Request archive” o “Request data”
  8. Esperar confirmación - X comenzará a compilar los datos

Desde Móvil (App iOS/Android)

  1. Abrir app de X
  2. Tap en ícono de perfil (esquina superior izquierda)
  3. Tap en “Settings and privacy”
  4. Tap en “Account”
  5. Bajar a sección “Data and permissions”
  6. Seleccionar “Your X data” (o “Your Twitter data”)
  7. Verificar identidad con código de email/teléfono
  8. Tap en “Request data”

⏱️ Tiempo de espera

Factor Tiempo estimado
Cuentas pequeñas (< 1,000 tweets) 2-12 horas
Cuentas medianas (1,000-10,000 tweets) 24-48 horas
Cuentas grandes (> 10,000 tweets) 48-72+ horas
Períodos de alta demanda Puede extenderse

Notificación: X envía un email cuando el archive está listo.

⚠️ Importante: El link de descarga expira en 7 días (algunas fuentes dicen 30, pero no confiar). Si expira, hay que solicitar de nuevo.

📥 Cómo descargar y abrir el archive

  1. Recibir email de X con asunto tipo “Your Twitter data is ready”
  2. Click en el link de descarga (asegurarse de estar logueado en X en ese navegador)
  3. Descargar ZIP a ubicación conocida
  4. Extraer ZIP con herramienta nativa del SO o 7-Zip/WinRAR
  5. Abrir Your_archive.html en cualquier navegador
  6. Navegar offline - el visor permite buscar, filtrar por fecha, ver media

🛠️ Método 2: Herramientas de Terceros (Para exportación rápida/CSV)

Circleboom (Recomendado - Official X Enterprise Developer)

URL: https://circleboom.com/twitter-management-tool/twitter-search-tool/export-tweets

Características: - ✅ Exportar tweets de cualquier cuenta pública (no solo la tuya) - ✅ Formato: CSV o Excel - ✅ Incluye: Tweet ID, texto, likes, retweets, replies, views, timestamp, hashtags, mentions, links, media status - ⚠️ Límite: 3,200 tweets más recientes (restricción de API de X) - ✅ Exportación inmediata (sin espera de 24-48h) - ✅ Official X Enterprise Developer (compliant con políticas de X)

Casos de uso: - Análisis competitivo de cuentas de otros usuarios - Exportación rápida para análisis en Excel/Google Sheets - Monitoreo de keywords en tiempo real - Backup parcial de cuenta propia

Precio: Tiene versión gratuita limitada y planes de pago

TweetDelete

URL: https://tweetdelete.net/

Características: - ✅ Enfocado en eliminar tweets masivamente - ✅ Puede exportar antes de borrar - ⚠️ Límite de tweets según plan - 💰 Versión gratuita limitada, pago para cuentas grandes

Casos de uso: - Limpieza de cuenta antes de exportar - Borrar tweets antiguos automáticamente

Archive Twitter (Twitter Archive Viewer)

URL: Varios (archive.twitter-downloader.com, etc.)

Características: - ✅ Visualización de archives existentes - ⚠️ Algunos son solo visores, no exportadores - ⚠️ Calidad variable, verificar reputación

Tweet Archivist (Analytics)

URL: https://www.tweetarchivist.com/

Características: - ✅ Analytics avanzado para archives - ✅ Dashboards interactivos - ✅ Sentiment analysis, trending topics - 💰 Servicio de pago

Casos de uso: - Análisis profundo de engagement - Identificar mejores tweets por rendimiento - Visualizaciones profesionales

⚠️ Limitaciones a Considerar

Del método oficial (X Archive)

Limitación Impacto Workaround
Media no siempre full-res Fotos/videos pueden ser thumbnails o links a t.co Descargar media por separado con herramientas como TweetSave
Engagement estático Métricas son snapshot al momento del archive, no se actualizan No hay workaround - es limitación de diseño
Followers con user IDs Difícil identificar quién es quién sin cross-reference Usar API de X o herramientas para resolver IDs a usernames
Sin analytics avanzados Solo datos crudos, sin gráficos ni insights Usar Python, Excel, o herramientas como Tweet Archivist
Búsqueda básica El visor HTML tiene búsqueda limitada Extraer tweets.js y analizar con scripts personalizados
Link expira en 7 días Si no descargas a tiempo, pierdes el archive Descargar inmediatamente al recibir notificación

De herramientas de terceros

Limitación Impacto
Límite de 3,200 tweets API de X solo permite acceder a tweets más recientes
No incluye tweets eliminados Solo puede exportar lo que aún existe en la plataforma
No incluye DMs o likes La mayoría solo exporta tweets públicos
Costo Las versiones completas suelen ser de pago
Riesgo de seguridad Algunas herramientas piden credenciales de X

🐍 Script Python para Procesar tweets.js

import json
import pandas as pd

# Leer tweets.js y quitar el wrapper de JavaScript
with open('data/tweets.js', 'r', encoding='utf-8') as f:
    content = f.read()
    # Quitar 'window.YTD.tweets.part0 = ' y brackets externos si existen
    json_str = content.split('=', 1)[1].strip()
    tweets = json.loads(json_str)

# Convertir a DataFrame
df = pd.json_normalize([tweet['tweet'] for tweet in tweets])

# Guardar como CSV
df.to_csv('tweets_export.csv', index=False, encoding='utf-8')

print(f"Exportados {len(df)} tweets a tweets_export.csv")
