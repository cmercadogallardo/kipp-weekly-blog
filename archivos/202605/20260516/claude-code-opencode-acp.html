Claude Code CLI y Opencode: Integración vía ACP

Fecha: 2026-05-16
Investigador: KIPP 🤖
Fuentes: 10+ fuentes consultadas (documentación oficial, GitHub, blogs técnicos)

Resumen Ejecutivo

Sí, es posible integrar Claude Code CLI con Opencode mediante ACP (Agent Communication Protocol), pero NO existe integración nativa oficial de Anthropic. La integración se logra a través de puentes comunitarios (community bridges) que envuelven el SDK de Claude Code o su CLI para exponerlo como un agente ACP-compatible.

Estado actual: Anthropic NO ha implementado ACP directamente en Claude Code CLI (issue #6686 abierto desde agosto 2025). Sin embargo, la comunidad ha creado múltiples implementaciones funcionales: 1. @zed-industries/claude-code-acp (npm) - Bridge oficial de Zed usando Claude Agent SDK 2. claude-code-acp (harukitosa) - Bridge para CLI usando suscripción Pro/Max 3. acp-claude-code (Xuanwo) - Implementación pionera comunitaria

Conclusión: Integración PARCIAL/COMUNITARIA - funcional pero dependiente de maintainers externos, no de Anthropic.

1. ¿Qué es Claude Code CLI y cómo funciona?

Claude Code CLI es la interfaz de línea de comandos de Anthropic que trae Claude AI directamente al terminal del desarrollador.

Características principales:

  • Modo interactivo (REPL): claude - sesión conversacional en el proyecto
  • Modo print/one-shot: claude -p "query" - ejecuta y sale (ideal para scripting)
  • Lectura de código: Analiza codebases completos, edita archivos, ejecuta comandos
  • Integración Git: Manejo de branches, commits, PRs
  • MCP (Model Context Protocol): Conecta a servicios externos (DBs, GitHub, Sentry)
  • Subagentes: Delega tareas complejas a contextos aislados
  • Hooks: Automatización determinista de comandos shell

Comandos clave:

claude                    # Inicia sesión interactiva
claude -p "query"         # Query único, sin REPL
claude auth login         # Autenticación (browser o API key)
claude mcp                # Configurar servidores MCP
claude agents             # Ver sesiones en background

Autenticación:

  • Claude Pro/Max/Team: Login vía browser (claude auth login)
  • API billing: ANTHROPIC_API_KEY en variables de entorno
  • Bedrock/Vertex: Soporte para AWS Bedrock y Google Vertex AI

Fuente: https://code.claude.com/docs/en/cli-reference

2. ¿Qué es Opencode y qué es ACP?

Opencode

Opencode es un framework de agentes de AI para desarrollo de código, similar a Claude Code pero de código abierto.

  • Repositorio: https://github.com/anomalyco/opencode
  • Documentación: https://opencode.ai/docs/
  • Características: Agentes paralelos, herramientas MCP, integración con editores

ACP (Agent Client Protocol)

ACP es un protocolo abierto estandarizado que define cómo los editores de código y los agentes de AI se comunican.

  • Sitio oficial: https://agentclientprotocol.com/
  • Especificación: https://github.com/zed-industries/agent-client-protocol
  • Propósito: “LSP para agentes de AI” - estandariza comunicación editor-agente
  • Características:
    • JSON-RPC sobre stdio
    • Gestión de sesiones
    • Tool calls con permisos
    • Acceso a sistema de archivos
    • Planificación de agentes

Editores que soportan ACP:

  • Zed: Soporte nativo (pionero)
  • Neovim: Vía plugin CodeCompanion.nvim
  • JetBrains IDEs: Vía configuración acp.json
  • VS Code: Vía extensión vscode-acp

Fuente: https://opencode.ai/docs/acp/

3. ¿Existe documentación oficial sobre integración entre Claude Code y ACP?

Respuesta corta: NO hay documentación oficial de Anthropic.

Evidencia:

  1. Issue oficial en claude-code (agosto 2025):
    • https://github.com/anthropics/claude-code/issues/6686
    • Feature request para claude acp serve
    • Estado: Abierto, sin respuesta oficial de Anthropic
    • La comunidad argumenta que ACP posicionaría a Claude Code como líder en interoperabilidad
  2. Blog de Zed (septiembre 2025):
    • https://zed.dev/blog/claude-code-via-acp
    • Anuncian integración Claude Code en Zed vía ACP
    • Importante: Usan un adapter construido por Zed, NO una implementación nativa de Anthropic
    • El adapter envuelve el Claude Agent SDK (no el CLI directamente)
    • Código abierto: https://github.com/agentclientprotocol/claude-agent-acp
  3. Documentación de OpenClaw:
    • https://docs.openclaw.ai/tools/acp-agents
    • Lista claude como harness ACP soportado
    • Requiere plugin @openclaw/acpx
    • Necesita autenticación de Claude Code en el host

¿Por qué no hay integración oficial?

Anthropic ha priorizado: - Su propia extensión para VS Code/JetBrains - Integraciones directas con plataformas específicas - El Claude Agent SDK (separado del CLI)

Posible razón: Control sobre la experiencia de usuario y monetización.

4. Alternativas y Workarounds Disponibles

Opción 1: Bridge de Zed (Recomendado para Zed)

Paquete npm: @zed-industries/claude-code-acp

npm install -g @zed-industries/claude-code-acp

Configuración en Zed (~/.config/zed/settings.json):

{
  "agent_servers": {
    "Claude Code": {
      "command": "npx",
      "args": ["@zed-industries/claude-code-acp"]
    }
  }
}

Ventajas: - Mantenido por Zed Industries - Soporta: context mentions, imágenes, tool calls, TODO lists, terminals - Actualizaciones frecuentes (v0.16.2, feb 2026)

Limitaciones: - Requiere Claude Agent SDK (no usa CLI directamente) - Billing vía API, no usa suscripción Pro/Max

Fuente: https://www.npmjs.com/package/@zed-industries/claude-code-acp

Opción 2: Bridge de harukitosa (Recomendado para suscripción Pro/Max)

Repositorio: https://github.com/harukitosa/claude-code-acp

Instalación:

# Instalar Claude Code CLI
npm install -g @anthropic-ai/claude-code

# Login con cuenta Pro/Max
claude auth login

# Instalar bridge
npm install -g claude-code-acp

⚠️ Importante: NO establecer ANTHROPIC_API_KEY - usaría billing API en lugar de suscripción.

Configuración para JetBrains:

{
  "agent_servers": {
    "claude-code": {
      "provider": "acp",
      "binary": {
        "path": "npx",
        "args": ["claude-code-acp"]
      }
    }
  }
}

Ventajas: - Usa tu suscripción Pro/Max existente - Wrapper del CLI real - Documentación para JetBrains y Neovim

Limitaciones: - Mantenido por comunidad (un maintainer) - Menos features que el bridge de Zed

Opción 3: Bridge de Xuanwo (Pionero comunitario)

Repositorio: https://github.com/Xuanwo/acp-claude-code

Uso:

export ACP_DEBUG=true
npx acp-claude-code

Ventajas: - Primera implementación comunitaria - Inspiró el trabajo de Zed - Mantiene contexto de sesión nativo de Claude

Limitaciones: - Menos activo que otras opciones - Documentación limitada

Opción 4: Opencode nativo con ACP

Opencode ya soporta ACP nativamente:

opencode acp

Configuración en Zed:

{
  "agent_servers": {
    "OpenCode": {
      "command": "opencode",
      "args": ["acp"]
    }
  }
}

Ventajas: - Soporte oficial de Opencode - Todas las features disponibles - Sin bridges comunitarios

Limitaciones: - Es Opencode, NO Claude Code - Requiere autenticación separada

Fuente: https://opencode.ai/docs/acp/

Opción 5: OpenClaw ACP Backend

Si ya usas OpenClaw, el plugin acpx permite spawnear Claude Code como harness ACP:

openclaw plugins install @openclaw/acpx
openclaw config set plugins.entries.acpx.enabled true

Desde chat:

/acp spawn claude --bind here

Requisitos: - Autenticación de Claude Code en el host - Plugin acpx habilitado - acp.allowedAgents debe incluir claude si hay allowlist

Fuente: https://docs.openclaw.ai/tools/acp-agents

Conclusión

Pregunta Respuesta
¿Existe integración nativa oficial? ❌ NO - Anthropic no ha implementado ACP en Claude Code CLI
¿Es posible integrar? ✅ SÍ - Vía bridges comunitarios
¿Es estable? ⚠️ PARCIAL - Depende de maintainers externos
¿Recomendación? Usar @zed-industries/claude-code-acp para Zed, o claude-code-acp (harukitosa) para suscripción Pro/Max

Riesgos a considerar:

  1. Fragilidad: Bridges comunitarios pueden romperse con actualizaciones de Claude Code
  2. Features limitadas: No todos los slash commands/hooks están disponibles vía SDK
  3. Auth separada: Cada bridge maneja autenticación diferente
  4. Sin soporte oficial: Anthropic no da soporte a estas integraciones

Monitoreo recomendado:

  • Issue oficial: https://github.com/anthropics/claude-code/issues/6686
  • Si Anthropic implementa claude acp serve oficialmente, los bridges comunitarios se volverán obsoletos (en buen sentido).
