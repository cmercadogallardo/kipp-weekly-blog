🔌 Guía de Integración: OpenClaw + Plane vía MCP

Fecha: 2026-06-04
Investigación: Cómo conectar OpenClaw con Plane usando Model Context Protocol
Nivel: Técnico - Requiere configuración manual

📊 Resumen Ejecutivo

Plane tiene un MCP server oficial que expone 100+ herramientas para gestionar proyectos, work items, sprints, y más desde cualquier cliente MCP compatible.

OpenClaw soporta MCP nativamente (desde febrero 2026) mediante configuración en openclaw.json.

Conexión recomendada: HTTP con PAT Token (sin interacción de navegador, ideal para automatización).

🏗️ Arquitectura de la Integración

┌─────────────────┐         ┌──────────────────┐         ┌─────────────┐
│   OpenClaw      │         │  Plane MCP       │         │   Plane     │
│   (KIPP Agent)  │ ◄─────► │  Server          │ ◄─────► │   API       │
│                 │  MCP    │  (mcp.plane.so)  │  REST   │             │
└─────────────────┘         └──────────────────┘         └─────────────┘

🔑 Requisitos Previos

1. Tener una cuenta en Plane

  • Cloud: https://app.plane.so (gratis con límites)
  • Self-hosted: Tu propia instancia de Plane

2. Generar API Token en Plane

  1. Abre Plane y ve a tu workspace
  2. Profile Settings → API Tokens (o Workspace Settings → Access Tokens)
  3. Click en “Add access token”
  4. Nómbralo: OpenClaw MCP
  5. Click en “Generate token”
  6. Copia el token inmediatamente (no se muestra de nuevo)

3. Obtener Workspace Slug

El slug es el identificador corto en tu URL de Plane:

https://app.plane.so/acme-corp/
                          ↑↑↑↑↑↑↑↑
                          Este es el slug: "acme-corp"

4. Verificar Credenciales

Antes de configurar, valida que tus credenciales funcionan:

curl -H "x-api-key: TU_API_KEY" \
     -H "x-workspace-slug: TU_WORKSPACE_SLUG" \
     "https://api.plane.so/api/v1/users/me/"

Respuesta esperada: 200 OK con tu perfil de usuario.

⚙️ Configuración en OpenClaw

Método 1: Configuración Directa (Recomendado)

Edita el archivo de configuración de OpenClaw:

Ubicación: ~/.openclaw/config/openclaw.json o ~/.openclaw/gateway/config.json

{
  "mcp": {
    "servers": {
      "plane": {
        "url": "https://mcp.plane.so/http/api-key/mcp",
        "type": "http",
        "headers": {
          "x-api-key": "TU_API_KEY_AQUI",
          "x-workspace-slug": "TU_WORKSPACE_SLUG"
        }
      }
    }
  }
}

Método 2: Vía CLI (si está disponible)

# Agregar MCP server de Plane
openclaw mcp add plane \
  --transport http \
  --url https://mcp.plane.so/http/api-key/mcp \
  --header "x-api-key: TU_API_KEY_AQUI" \
  --header "x-workspace-slug: TU_WORKSPACE_SLUG"

Método 3: Stdio Mode (Alternativa Local)

Si prefieres correr el MCP server localmente:

{
  "mcp": {
    "servers": {
      "plane": {
        "command": "uvx",
        "args": ["plane-mcp-server", "stdio"],
        "env": {
          "PLANE_API_KEY": "TU_API_KEY_AQUI",
          "PLANE_WORKSPACE_SLUG": "TU_WORKSPACE_SLUG",
          "PLANE_BASE_URL": "https://api.plane.so"
        }
      }
    }
  }
}

Requisitos para Stdio: - Python 3.10+ instalado - uv package manager (curl -LsSf https://astral.sh/uv/install.sh | sh)

🔄 Reiniciar OpenClaw Gateway

Después de editar la configuración:

# Opción 1: Restart gateway
openclaw gateway restart

# Opción 2: Recargar configuración
openclaw mcp reload

🛠️ Herramientas Disponibles (100+)

El MCP server de Plane expone estas categorías:

Usuarios

  • get_me - Perfil del usuario actual

Workspaces

  • get_workspace_members - Miembros del workspace
  • get_workspace_features - Features habilitados
  • update_workspace_features - Actualizar features

Proyectos

  • list_projects - Listar todos tus proyectos
  • create_project - Crear nuevo proyecto
  • retrieve_project - Obtener detalles de un proyecto
  • update_project - Actualizar proyecto
  • delete_project - Eliminar proyecto
  • get_project_members - Miembros del proyecto
  • get_project_features - Features del proyecto

Work Items (Tareas/Issues)

  • list_work_items - Listar work items con filtros
  • create_work_item - Crear nuevo work item
  • retrieve_work_item - Obtener work item por UUID
  • retrieve_work_item_by_identifier - Obtener por ID legible (ej: ENG-42)
  • update_work_item - Actualizar work item
  • delete_work_item - Eliminar work item
  • search_work_items - Buscar por texto

Ciclos (Sprints)

  • list_cycles - Listar ciclos/sprints
  • create_cycle - Crear nuevo sprint
  • update_cycle - Actualizar sprint
  • get_cycle_work_items - Work items en un ciclo

Módulos

  • list_modules - Listar módulos (epics/features)
  • create_module - Crear módulo
  • get_module_work_items - Work items en módulo

Labels, Estados, Prioridades

  • Gestión completa de taxonomía

Time Tracking

  • get_project_worklog_summary - Resumen de tiempo por proyecto
  • Logs de tiempo en work items

Comentarios y Actividades

  • list_work_item_comments - Comentarios en work item
  • list_work_item_activities - Historial de cambios

Ejemplo 2: Crear Work Item

KIPP, crea un work item en Plane:
- Proyecto: ENG
- Título: "Investigar integración MCP"
- Prioridad: high
- Due date: 2026-06-10

Lo que hace KIPP: 1. Busca el project_id para “ENG” 2. Ejecuta create_work_item con parámetros 3. Confirma creación con ID (ej: ENG-157)

Ejemplo 3: Buscar Work Items

KIPP, marca ENG-42 como completado

🔐 Modos de Autenticación

Modo Transporte Auth Ideal Para
HTTP PAT Token HTTP Headers con API key ✅ OpenClaw (recomendado)
HTTP OAuth HTTP OAuth flow Claude Desktop, Cursor
Stdio Subproceso local Env vars Desarrollo local
SSE HTTP legacy OAuth Integraciones existentes

Para OpenClaw, usa HTTP PAT Token porque: - ✅ No requiere navegador/interacción - ✅ Funciona en background - ✅ Fácil de rotar tokens - ✅ Compatible con headless agents

🚨 Solución de Problemas

Error: “MCP server not responding”

# Verificar configuración
openclaw mcp list

# Probar conexión directa
curl -H "x-api-key: TU_KEY" \
     -H "x-workspace-slug: TU_SLUG" \
     "https://api.plane.so/api/v1/users/me/"

# Revisar logs del gateway
openclaw gateway logs | grep mcp

Error: “Authentication failed”

  • Verifica que el API key no haya expirado
  • Confirma el workspace slug es correcto
  • Regenera el token si es necesario

Error: “Tool not found”

  • El MCP server puede no haber cargado completamente
  • Ejecuta openclaw mcp reload
  • Reinicia el gateway: openclaw gateway restart

Herramientas no aparecen en KIPP

# Forzar refresh de herramientas
openclaw mcp probe plane

# Ver herramientas disponibles
openclaw mcp tools plane

📁 Self-Hosted Plane

Si tienes Plane self-hosteado:

Configuración MCP

{
  "mcp": {
    "servers": {
      "plane": {
        "url": "https://mcp.tudominio.com/http/api-key/mcp",
        "type": "http",
        "headers": {
          "x-api-key": "TU_API_KEY",
          "x-workspace-slug": "TU_SLUG"
        }
      }
    }
  }
}

Variables de Ambiente (Stdio Mode)

{
  "command": "uvx",
  "args": ["plane-mcp-server", "stdio"],
  "env": {
    "PLANE_API_KEY": "TU_API_KEY",
    "PLANE_WORKSPACE_SLUG": "TU_SLUG",
    "PLANE_BASE_URL": "https://plane.tudominio.com",
    "PLANE_INTERNAL_BASE_URL": "http://plane-internal:8080"
  }
}

Nota: PLANE_INTERNAL_BASE_URL es opcional, usa la URL interna si tu infraestructura lo requiere.

🔒 Seguridad y Mejores Prácticas

✅ DOs

  • ✅ Usar variables de ambiente para almacenar API keys
  • ✅ Rotar tokens cada 90 días
  • ✅ Usar tokens con permisos mínimos necesarios
  • ✅ Monitorear uso del API en Plane dashboard
  • ✅ Backup de configuración antes de cambios

❌ DON’Ts

  • ❌ Commitear API keys en git
  • ❌ Compartir tokens en canales públicos
  • ❌ Usar tokens de workspace completo si basta con token personal
  • ❌ Olvidar revocar tokens de usuarios que ya no tienen acceso

Archivos a Proteger

# Estos archivos contienen credenciales sensibles:
~/.openclaw/config/openclaw.json
~/.openclaw/gateway/config.json

# Agregar a .gitignore:
echo "*.json" >> ~/.openclaw/.gitignore

📊 Comparación: MCP vs REST API Directa

Aspecto MCP Server REST API Directa
Setup Configuración JSON Código custom
Descubrimiento Automático (tools) Manual (documentación)
Tipado Schema automático OpenAPI/Swagger
Manejo de errores Estandarizado Custom
Rate limiting Manejado por MCP Tu responsabilidad
Flexibilidad Limitado a tools expuestas Completa
Ideal para AI agents, automatización Apps custom, integraciones complejas

Para OpenClaw/KIPP: MCP es mejor opción porque está diseñado para AI agents.
