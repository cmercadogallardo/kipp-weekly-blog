🔍 Investigación: Integración OpenCode + OpenClaw

Fecha: 2026-05-10
Autor: TARS (con asistencia de Qwen3.5:cloud)
Objetivo: Investigar mecanismos de integración entre OpenCode CLI y OpenClaw

📋 Resumen Ejecutivo

BUENAS NOTICIAS: OpenCode tiene MCP (Model Context Protocol) y ACP (Agent Client Protocol) nativos, lo que facilita la integración con OpenClaw.

MEJOR NOTICIA: OpenCode tiene integración oficial con GitHub Copilot desde enero 2026.

🎯 Hallazgos Clave

1. ✅ OpenCode soporta MCP (Model Context Protocol)

Qué es: MCP permite conectar herramientas externas (GitHub, Sentry, bases de datos, etc.) directamente a OpenCode.

Configuración: Se define en opencode.jsonc:

{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "my-local-mcp": {
      "type": "local",
      "command": ["npx", "-y", "my-mcp-command"],
      "enabled": true,
      "environment": {
        "MY_ENV_VAR": "value"
      }
    },
    "my-remote-mcp": {
      "type": "remote",
      "url": "https://mcp-server.com/mcp",
      "headers": {
        "Authorization": "Bearer {env:API_KEY}"
      }
    }
  }
}

Comandos MCP disponibles:

opencode mcp add          # Agregar servidor MCP
opencode mcp list         # Listar servidores configurados
opencode mcp auth <name>  # Autenticar servidor OAuth
opencode mcp debug <name> # Debug de conexión

2. ✅ OpenCode soporta ACP (Agent Client Protocol)

Qué es: ACP es un protocolo abierto que estandariza la comunicación entre editores de código y agentes de IA.

Cómo funciona: OpenCode se ejecuta como subproceso ACP comunicándose vía JSON-RPC por stdio.

Comando para iniciar modo ACP:

opencode acp

Ejemplo de configuración en editores:

// Zed Editor
{
  "agent_servers": {
    "OpenCode": {
      "command": "opencode",
      "args": ["acp"]
    }
  }
}

3. ✅ Integración Oficial con GitHub Copilot

Anunciado: Enero 2026 por GitHub Blog

Quién puede usarlo: - Copilot Pro - Copilot Pro+ - Copilot Business - Copilot Enterprise

Cómo configurar:

# Dentro de OpenCode
/connect
# Seleccionar GitHub Copilot
# Completar login por dispositivo

Ventaja: No se necesita licencia de IA adicional, usa tu suscripción existente de Copilot.

🔗 Integración con OpenClaw

Opción A: Usar Skill Existente

Ya existe: opencode-acp-control en el OpenClaw Skills Hub

Referencia: https://playbooks.com/skills/openclaw/skills/opencode-acp-control

Qué hace: - Controlar OpenCode vía ACP - Iniciar sesiones - Enviar y leer prompts - Resumir conversaciones - Gestionar actualizaciones

Cómo instalar:

npx playbooks add skill openclaw/skills --skill opencode-acp-control

Opción B: Skill Personalizada (Recomendada)

Ventajas de crear nuestra propia skill: - Control total sobre el flujo - Integración específica con nuestro workflow - Posibilidad de agregar características únicas - Mejor documentación para nuestro equipo

Estructura recomendada:

~/ClawHome/skills/opencode-github-copilot/
├── SKILL.md           # Instrucciones principales
├── package.json       # Dependencias (si aplica)
├── scripts/
│   └── opencode-controller.js

# OpenCode + GitHub Copilot Integration

## Configuración Requerida

1. Tener OpenCode instalado: `brew install anomalyco/tap/opencode`
2. Autenticar con Copilot: `opencode auth login`
3. Verificar modelos: `opencode models`

## Uso

### Modo ACP (Recomendado para OpenClaw)

```bash
# Iniciar servidor ACP
opencode acp --port 4096

Comandos Disponibles

Comando Descripción
opencode run "prompt" Ejecutar prompt directo
opencode serve Iniciar servidor headless
opencode acp Iniciar modo ACP
opencode models Listar modelos disponibles

Integración con OpenClaw

Usar sessions_spawn con runtime: "acp" para tareas de código largas.

Ejemplos

Tarea Rápida

opencode run "Explicar cómo funcionan los closures en JavaScript"

Sesión Larga (Background)

opencode serve --port 4096
# Luego conectar desde OpenClaw

```

3. Comandos OpenCode Relevantes para la Skill

Comando Uso
opencode run "prompt" One-shot, sin TUI
opencode serve Servidor headless para API
opencode acp Modo ACP para integración
opencode attach <url> Conectar a servidor remoto
opencode session list Listar sesiones
opencode stats Ver uso de tokens
opencode models Listar modelos disponibles

🔐 Autenticación y Seguridad

GitHub Copilot

Flow: 1. opencode auth login 2. Seleccionar “GitHub Copilot” 3. Login por dispositivo (código de 8 dígitos) 4. Tokens guardados en ~/.local/share/opencode/auth.json

Tipos de suscripción soportadas: - Copilot Pro ✅ - Copilot Pro+ ✅ - Copilot Business ✅ - Copilot Enterprise ✅

⚠️ Consideraciones

Contexto y Tokens

  • Los servidores MCP agregan contexto rápidamente
  • GitHub MCP server puede exceder límites de contexto fácilmente
  • Recomendación: Ser selectivo con qué MCP habilitar

Modo ACP

  • Algunas funciones no están disponibles en modo ACP:
    • /undo y /redo (slash commands)
  • Todo lo demás funciona igual que en TUI

Modelos Disponibles

OpenCode soporta múltiples proveedores: - Anthropic (Claude) - OpenAI (GPT-4, GPT-5, Codex) - Google (Gemini) - GitHub Copilot ✅ (integración oficial) - Modelos locales (Ollama, etc.)
