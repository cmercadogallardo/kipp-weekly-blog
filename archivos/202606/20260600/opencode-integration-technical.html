🔌 Integración Técnica: OpenClaw ↔︎ OpenCode

Fecha: 2026-05-10
Autor: TARS
Propósito: Documentar el mecanismo técnico de comunicación entre OpenClaw y OpenCode para CASE

📊 Arquitectura de Comunicación

┌─────────────────────────────────────────────────────────────┐
│                    OPENCLAW (Discord)                       │
│  - sessions_spawn(runtime="subagent")                       │
│  - subagents: action=list/steer/kill                        │
│  - sessions_send (notificaciones)                           │
└────────────────────┬────────────────────────────────────────┘
                     │
                     │ sessions_spawn()
                     │ task: "<prompt para OpenCode>"
                     │ cwd: "<directorio del proyecto>"
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│              SUB-AGENTE (OpenClaw runtime)                  │
│  - Lee SKILL.md de opencode-github-copilot                  │
│  - Ejecuta comandos opencode vía exec tool                  │
│  - Monitorea progreso con process tool                      │
│  - Notifica completado con openclaw system event            │
└────────────────────┬────────────────────────────────────────┘
                     │
                     │ exec tool
                     │ command: "opencode run <prompt>"
                     │ pty: false (one-shot)
                     │ background: true (tareas largas)
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│                    OPENCODE CLI                             │
│  - Carga auth desde ~/.local/share/opencode/auth.json       │
│  - Conecta a GitHub Copilot API                             │
│  - Ejecuta modelo (gpt-5.2-codex, claude-sonnet, etc.)     │
│  - Ejecuta herramientas (file edit, bash, etc.)             │
└────────────────────┬────────────────────────────────────────┘
                     │
                     │ HTTPS API
                     │ OAuth token
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│                  GITHUB COPILOT API                         │
│  - Modelos: GPT-5, Claude, Gemini, etc.                     │
│  - Context window: hasta 200K tokens                        │
│  - Tools: file edit, bash, grep, etc.                       │
└─────────────────────────────────────────────────────────────┘

🛠️ Mecanismo de Invocación

Nivel 1: OpenClaw → Sub-Agente

Herramienta: sessions_spawn

// En OpenClaw (TypeScript)
const result = await sessions_spawn({
  runtime: "subagent",
  label: "opencode-feature-build",
  task: `
    Vas a implementar una feature usando OpenCode + GitHub Copilot.
    
    CONTEXTO:
    - Proyecto: /Users/cristianmercado/ClawHome/repositories/flair-dashboard
    - Especificación: ./documentation/WORKFLOW/READY/flair-invoice-pdf.md
    
    TAREA:
    1. Lee la especificación completa
    2. Implementa el código necesario
    3. Ejecuta tests
    4. Haz commit y push
    5. Notifica con: openclaw system event --text "Done" --mode now
    
    Usa OpenCode CLI para todo el desarrollo.
  `,
  cwd: "/Users/cristianmercado/ClawHome/repositories/flair-dashboard"
});

Parámetros clave: | Parámetro | Tipo | Requerido | Descripción | |———–|——|———–|————-| | runtime | string | ✅ | "subagent" para sub-agente OpenClaw | | label | string | ❌ | Nombre descriptivo para monitoreo | | task | string | ✅ | Prompt completo con contexto y tarea | | cwd | string | ❌ | Directorio de trabajo (proyecto) | | model | string | ❌ | Override de modelo (default: qwen3.5:cloud) |

Respuesta:

{
  "status": "accepted",
  "childSessionKey": "agent:main:subagent:xxx-xxx-xxx",
  "runId": "yyy-yyy-yyy",
  "mode": "run",
  "note": "Auto-announce is push-based..."
}

Nivel 2: Sub-Agente → OpenCode CLI

Herramienta: exec (bash)

// En el sub-agente (leyendo SKILL.md)
const execResult = await exec({
  command: `opencode run "Implementar componente de facturación PDF"`,
  workdir: "/Users/cristianmercado/ClawHome/repositories/flair-dashboard",
  background: true,  // Para tareas largas
  timeout: 1800      // 30 minutos
});

Para tareas one-shot (rápidas):

# Sin background, espera completado
opencode run "Explicar qué es un closure"

Para tareas largas (background):

# Con background, retorna sessionId
opencode run --background "Refactorizar módulo completo"

Nivel 3: OpenCode CLI → GitHub Copilot

Configuración: ~/.local/share/opencode/auth.json

{
  "github-copilot": {
    "type": "oauth",
    "token: [REDACTED],
    "expiresAt": "2026-06-10T00:00:00Z"
  }
}

Comando interno de OpenCode:

# OpenCode internally hace:
curl -X POST https://api.githubcopilot.com/chat/completions \
  -H "Authorization: Bearer gho_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.2-codex",
    "messages": [...],
    "tools": [...]
  }'

📡 Flujo de Notificaciones

Completado Exitoso

1. Sub-agente termina tarea
2. Ejecuta: openclaw system event --text "Done: <resumen>" --mode now
3. OpenClaw recibe evento
4. OpenClaw notifica al usuario en Discord
5. Usuario revisa cambios

Ejemplo de notificación:

✅ **Tarea Completada**

**Sub-agente:** opencode-feature-build
**Tarea:** Implementar componente de facturación PDF
**Tiempo:** 12m 34s
**Cambios:** 15 archivos modificados
**Commit:** a1b2c3d - "feat: invoice PDF component"

**Archivos:**
- web/src/app/features/invoice/...
- functions/src/invoice/...

**Review:** https://github.com/.../commit/a1b2c3d

Error o Fallo

1. Sub-agente detecta error
2. Ejecuta: openclaw system event --text "Error: <descripción>" --mode now
3. OpenClaw notifica al usuario
4. Usuario decide retry o abortar

Ejemplo de notificación de error:

❌ **Error en Tarea**

**Sub-agente:** opencode-feature-build
**Error:** ng build failed - 3 compilation errors
**Archivos con error:**
- invoice.component.ts (línea 45)
- invoice.service.ts (línea 23)

**Logs:**

Error: Cannot find module ‘@angular/core’ …


**¿Reintentar o abortar?**

🔧 Comandos OpenCode Disponibles

Comandos para CASE

Comando Uso Ejemplo
opencode run "<prompt>" Tarea one-shot opencode run "Crear componente"
opencode run --model <model> "<prompt>" Modelo específico opencode run --model github-copilot/gpt-5.2-codex "Optimizar"
opencode run --background "<prompt>" Background mode opencode run --background "Refactorizar"
opencode session list Listar sesiones opencode session list -n 5
opencode session log <id> Ver logs opencode session log abc123
opencode stats Ver uso opencode stats --days 7
opencode models Listar modelos opencode models github-copilot

Flags Útiles

Flag Descripción Ejemplo
--model Modelo específico --model github-copilot/claude-sonnet-4.5
--background Ejecutar en background --background
--agent Agente específico --agent plan (solo lectura)
--permission-mode Control de permisos --permission-mode bypassPermissions
--continue Continuar sesión --continue
--session Session ID --session abc123

📝 Ejemplo Completo: CASE Implementa Feature

Paso 1: OpenClaw Spawnea Sub-Agente

// OpenClaw recibe mensaje de Cristian:
// "CASE, implementa la especificación flair-invoice-pdf"

// OpenClaw ejecuta:
sessions_spawn({
  runtime: "subagent",
  label: "case-flair-invoice",
  task: `
    Eres CASE, agente de desarrollo de BSD Enterprise.
    
    TAREA: Implementar especificación "flair-invoice-pdf"
    
    CONTEXTO:
    - Especificación: ./documentation/WORKFLOW/READY/flair-invoice-pdf.md
    - Proyecto: /Users/cristianmercado/ClawHome/repositories/flair-dashboard
    
    PROCESO:
    1. Lee la especificación completa
    2. Analiza código existente
    3. Implementa la feature con OpenCode + GitHub Copilot
    4. Ejecuta tests: npm test
    5. Valida build: ng build
    6. Commit y push: git add/commit/push
    7. Notifica: openclaw system event --text "Done" --mode now
    
    MODELO: github-copilot/gpt-5.2-codex
  `,
  cwd: "/Users/cristianmercado/ClawHome/repositories/flair-dashboard"
});

Paso 2: Sub-Agente Lee SKILL.md

# El sub-agente lee: ~/ClawHome/skills/opencode-github-copilot/SKILL.md

# Extrae:
- Comandos disponibles
- Patrones de uso
- Mejores prácticas
- Troubleshooting

Paso 3: Sub-Agente Ejecuta OpenCode

# Lee especificación
cat ./documentation/WORKFLOW/READY/flair-invoice-pdf.md

# Ejecuta OpenCode para implementar
opencode run "Implementar componente de facturación PDF según especificación.
Incluir:
- Componente Angular con PrimeNG
- Servicio para generar PDF
- Cloud Function para procesamiento
- Tests unitarios"

# Espera completado (background mode)
process action:poll sessionId:xxx

# Revisa logs
process action:log sessionId:xxx

Paso 4: Sub-Agente Valida

# Ejecuta tests
npm test

# Valida build
ng build

# Si hay errores, los reporta:
openclaw system event --text "Error: ng build failed" --mode now

Paso 5: Sub-Agente Hace Commit

# Git operations
git add .
git commit -m "feat: invoice PDF component with PrimeNG

- Componente con tabla y filtros
- Servicio para generación de PDF
- Cloud Function para procesamiento
- Tests unitarios

Spec: flair-invoice-pdf.md"
git push origin main

Paso 6: Sub-Agente Notifica

# Notificación final
openclaw system event --text "Done: Invoice PDF component implementado (15 archivos, 2h)" --mode now

Paso 7: OpenClaw Notifica a Cristian

✅ **CASE Completó la Tarea**

**Feature:** Invoice PDF Component
**Tiempo:** 2h 15m
**Archivos:** 15 modificados
**Commit:** a1b2c3d

🔐 Consideraciones de Seguridad

Auth Tokens

Ubicación Tipo Protección
~/.local/share/opencode/auth.json OAuth tokens 600 (solo usuario)
~/.config/opencode/opencode.jsonc Config MCP 644 (lectura usuario)
Environment variables API keys Proceso del usuario

Permisos de Archivo

# Verificar permisos
ls -la ~/.local/share/opencode/auth.json
# -rw-------  1 cristianmercado  staff  ...

# Si no son correctos:
chmod 600 ~/.local/share/opencode/auth.json

Sandboxing

OpenCode ejecuta en el contexto del usuario: - ✅ Acceso completo al home del usuario - ✅ Acceso a repos en ~/ClawHome/repositories/ - ❌ No acceso a system files (/System, /etc) - ❌ No acceso a otros usuarios

🚨 Troubleshooting Técnico

Error: “Provider not found”

Causa: Auth token expiró o no está configurado

Solución:

# Verificar auth
opencode auth list

# Si está vacío o expirado:
opencode auth login

Error: “Context limit exceeded”

Causa: Prompt + contexto excede límite del modelo

Solución:

# Reducir contexto
# O usar modelo con mayor contexto:
opencode run --model github-copilot/gpt-5.2-codex "Prompt más corto"

Error: “Permission denied”

Causa: OpenCode intenta editar archivo sin permisos

Solución:

# Opción 1: Dar permisos explícitos en prompt
opencode run "Editar archivo X. Permiso concedido."

# Opción 2: Bypass (solo dev)
opencode run --permission-mode bypassPermissions "Prompt"

Sub-agente se cuelga

Causa: OpenCode en modo interactivo o esperando input

Solución:

# Matar proceso OpenCode
killall opencode

# Matar sub-agente
subagents action:kill target:xxx

# Reintentar con background mode
opencode run --background "Prompt"
