📋 Resumen Ejecutivo

Qué es: Integración de OpenClaw con OpenCode CLI usando GitHub Copilot como proveedor de modelos.

Por qué: Permite a agentes (CASE, TARS, etc.) usar modelos de última generación (GPT-5, Claude, etc.) con tu suscripción existente de GitHub Copilot.

Estado: ✅ OPERATIVO - Skill creada y testeada exitosamente

🔧 Instalación y Setup

Paso 1: Instalar OpenCode

# macOS (Homebrew)
brew install anomalyco/tap/opencode

# Verificar instalación
opencode --version

Paso 2: Autenticar con GitHub Copilot

# Iniciar login
opencode auth login

# Seguir instrucciones:
# 1. Seleccionar "GitHub Copilot" como proveedor
# 2. Obtener código de 8 dígitos
# 3. Visitar https://github.com/login/device
# 4. Ingresar código y autorizar

Paso 3: Verificar Autenticación

# Listar credenciales
opencode auth list

# Debe mostrar:
# ● GitHub Copilot oauth

Paso 4: Ver Modelos Disponibles

# Listar todos los modelos
opencode models --refresh

# Filtrar por GitHub Copilot
opencode models github-copilot

Modelos disponibles: - github-copilot/gpt-5.2-codex ⭐ (default para código) - github-copilot/gpt-5.3-codex - github-copilot/gpt-5.4 - github-copilot/claude-sonnet-4.5 - github-copilot/gpt-4o - github-copilot/gemini-2.5-pro

📖 Comandos Básicos

Tarea Rápida (One-Shot)

# Pregunta conceptual
opencode run "Explicar qué es un closure en JavaScript"

# Generar código
opencode run "Crear función para validar email en TypeScript"

# Con modelo específico
opencode run --model github-copilot/gpt-5.2-codex "Optimizar esta query de Firestore"

Tarea con Contexto de Proyecto

# Navegar al proyecto
cd /path/to/project

# Ejecutar con contexto del directorio
opencode run "Agregar validación de email al formulario de login"

Tarea Larga (Background)

# Iniciar en background
opencode run --background "Refactorizar módulo de autenticación"

# Ver progreso
opencode session list

# Ver logs
opencode session log <sessionId>

Permisos de OpenCode

OpenCode pide confirmación para: - ✏️ Editar archivos fuera del directorio de trabajo - 🔧 Ejecutar comandos shell - 🌐 Acceder a URLs externas - 📁 Leer archivos sensibles

Bypass de Permisos (Solo Desarrollo)

# ⚠️ SOLO en entornos seguros
opencode run --permission-mode bypassPermissions "Tu prompt"

Advertencia: No usar en producción o con código no confiable.

🚨 Solución de Problemas

Error: “No authenticated providers”

# Verificar autenticación
opencode auth list

# Si está vacío,重新 autenticar
opencode auth login

Error: “Model not found”

# Refresh de modelos
opencode models --refresh

# Verificar nombre del modelo
opencode models github-copilot

Error: “Context limit exceeded”

  • Reducir alcance del prompt
  • Dividir tarea en múltiples prompts más pequeños
  • Usar agente plan para análisis primero

OpenCode se cuelga

# Listar procesos
ps aux | grep opencode

# Matar proceso
killall opencode

# Reiniciar
opencode auth list  # Verifica que sigue autenticado

🎓 Mejores Prácticas

✅ DOs

  • ✅ Ser específico en los prompts
  • ✅ Incluir contexto relevante (archivos, errores)
  • ✅ Dividir tareas grandes en pasos
  • ✅ Revisar código generado antes de commit
  • ✅ Usar agente plan para análisis exploratorio
  • ✅ Monitorear uso de tokens

❌ DON’Ts

  • ❌ Prompts vagos (“arregla todo”)
  • ❌ No revisar código antes de usar
  • ❌ Ejecutar en producción sin testing
  • ❌ Ignorar warnings de permisos
  • ❌ Usar --yolo en código crítico

🔄 Integración con OpenClaw

Cómo OpenClaw Usa la Skill

OpenClaw automáticamente usa la skill cuando: 1. Detecta una tarea de código larga (>2-3 min) 2. El usuario pide explícitamente usar OpenCode 3. Se requiere contexto de proyecto extenso

Comando para Spawnear Sub-Agente

// OpenClaw usa sessions_spawn internamente
sessions_spawn({
  runtime: "subagent",
  label: "opencode-<tarea>",
  task: "<descripción clara de la tarea>",
  cwd: "~/ClawHome/repositories/<proyecto>"
})

Notificación de Completado

Agregar al prompt:

Cuando termines, ejecuta:
openclaw system event --text "Completado: [resumen]" --mode now

🎯 Workflow para CASE (Offline Mode)

Cuando CASE Trabaja Offline

  1. Leer especificación del BACKLOG
  2. Planear implementación (agente plan)
  3. Ejecutar con OpenCode (agente build)
  4. Validar cambios (tests, build)
  5. Commit y push a main
  6. Notificar a OpenClaw

Ejemplo de Prompt para CASE

Eres CASE, agente de desarrollo de BSD Enterprise.

Tu tarea: Implementar la especificación <nombre-spec>

Proceso:
1. Lee la especificación en ./documentation/WORKFLOW/BACKLOG/
2. Analiza el código existente
3. Implementa los cambios necesarios
4. Ejecuta tests y valida el build
5. Haz commit y push a main
6. Notifica a OpenClaw cuando termines

Usa OpenCode con GitHub Copilot para todo el desarrollo.
Modelo recomendado: github-copilot/gpt-5.2-codex
