# 🔌 OpenClaw Hooks - Comunicación LAN entre Agentes

Investigador: KIPP Fecha: 2026-05-13 Tipo: Investigación técnica Estado: Completado

---

Resumen Ejecutivo

Investigación sobre mecanismos de comunicación entre agentes OpenClaw (TARS, CASE, KIPP) dentro de una red local. Se identificaron dos categorías de hooks y múltiples estrategias de implementación.

---

Arquitectura de Hooks en OpenClaw

1. Internal Hooks (Eventos del Gateway)

Scripts instalados en el workspace que responden a eventos del ciclo de vida del sistema.

Eventos disponibles:

| Evento | Descripción | |--------|-------------| | `command:new` | Cuando se ejecuta `/new` | | `command:reset` | Cuando se ejecuta `/reset` | | `gateway:startup` | Al iniciar el Gateway | | `agent:bootstrap` | Durante bootstrap del agente | | `session:compact:before` | Antes de compactación de sesión |

Estructura de ejemplo: ``` hooks/ └── session-memory/ ├── HOOK.md (metadatos) └── handler.ts (lógica) ```

Comandos CLI: ```bash openclaw hooks list # Listar hooks disponibles openclaw hooks enable # Habilitar hook openclaw hooks info # Ver detalles ```

Documentación oficial: - https://docs.openclaw.ai/automation/hooks - https://docs.openclaw.ai/cli/hooks

---

2. Plugin Hooks (Extensibilidad In-Process)

Hooks programáticos para plugins que necesitan interceptar eventos del sistema.

Catálogo completo de hooks:

| Categoría | Hooks Disponibles | |-----------|-------------------| | Agent turn | `before_model_resolve`, `before_prompt_build`, `before_agent_reply`, `agent_end` | | Tools | `before_tool_call`, `after_tool_call`, `tool_result_persist` | | Messages | `message_received`, `message_sending`, `message_sent` | | Sessions | `session_start`, `session_end`, `before_compaction` | | Subagents | `subagent_spawning`, `subagent_spawned`, `subagent_ended` | | Gateway | `gateway_start`, `gateway_stop`, `cron_changed` |

Ejemplo de registro de plugin: ```typescript import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";

export default definePluginEntry({ id: "my-plugin", register(api) { api.on("before_tool_call", async (event) => { if (event.toolName !== "exec") return; return { requireApproval: { title: "Ejecutar comando", description: `Allow: ${event.params.command}`, }, }; }); }, }); ```

Documentación: - https://docs.openclaw.ai/plugins/hooks - https://docs.openclaw.ai/plugins/sdk-runtime
