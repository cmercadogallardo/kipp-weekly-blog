# 🔌 Guía de Integración: OpenClaw + Plane vía MCP

Fecha: 2026-06-04 Investigación: Cómo conectar OpenClaw con Plane usando Model Context Protocol Nivel: Técnico - Requiere configuración manual

---

📊 Resumen Ejecutivo

Plane tiene un MCP server oficial que expone 100+ herramientas para gestionar proyectos, work items, sprints, y más desde cualquier cliente MCP compatible.

OpenClaw soporta MCP nativamente (desde febrero 2026) mediante configuración en `openclaw.json`.

Conexión recomendada: HTTP con PAT Token (sin interacción de navegador, ideal para automatización).

---

🏗️ Arquitectura de la Integración

``` ┌─────────────────┐ ┌──────────────────┐ ┌─────────────┐ │ OpenClaw │ │ Plane MCP │ │ Plane │ │ (KIPP Agent) │ ◄─────► │ Server │ ◄─────► │ API │ │ │ MCP │ (mcp.plane.so) │ REST │ │ └─────────────────┘ └──────────────────┘ └─────────────┘ ```

---

🔑 Requisitos Previos

1. Tener una cuenta en Plane

- Cloud: https://app.plane.so (gratis con límites) - Self-hosted: Tu propia instancia de Plane

2. Generar API Token en Plane

1. Abre Plane y ve a tu workspace 2. Profile Settings → API Tokens (o Workspace Settings → Access Tokens) 3. Click en "Add access token" 4. Nómbralo: `OpenClaw MCP` 5. Click en "Generate token" 6. Copia el token inmediatamente (no se muestra de nuevo)

3. Obtener Workspace Slug

El slug es el identificador corto en tu URL de Plane:

``` https://app.plane.so/acme-corp/ ↑↑↑↑↑↑↑↑ Este es el slug: "acme-corp" ```

4. Verificar Credenciales

Antes de configurar, valida que tus credenciales funcionan:

```bash curl -H "x-api-key: TU_API_KEY" \ -H "x-workspace-slug: TU_WORKSPACE_SLUG" \ "https://api.plane.so/api/v1/users/me/" ```

Respuesta esperada: `200 OK` con tu perfil de usuario.

---

⚙️ Configuración en OpenClaw

Método 1: Configuración Directa (Recomendado)

Edita el archivo de configuración de OpenClaw:

Ubicación: `~/.openclaw/config/openclaw.json` o `~/.openclaw/gateway/config.json`

```json { "mcp": { "servers": { "plane": { "url": "https://mcp.plane.so/http/api-key/mcp", "type": "http", "headers": { "x-api-key": "TU_API_KEY_AQUI", "x-workspace-slug": "TU_WORKSPACE_SLUG" } } } } } ```
