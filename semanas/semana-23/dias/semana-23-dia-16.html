# Fireflies.ai MCP + NotebookLM Integration Research

Investigación realizada por: KIPP Fecha: 2026-05-14 Estado: Completado Iteraciones de validación: 3 fuentes contrastadas por tema

---

📋 Resumen Ejecutivo

Esta investigación contrasta las posibilidades de integración entre Fireflies.ai y Google NotebookLM mediante Model Context Protocol (MCP) y APIs oficiales/comunitarias.

---

1️⃣ MCP para Fireflies.ai

✅ MCP OFICIAL DISPONIBLE

Fireflies.ai lanzó su MCP Server oficial en beta en marzo de 2026.

Detalles clave:

| Aspecto | Información | |---------|-------------| | URL del servidor MCP | `https://api.fireflies.ai/mcp` | | Estado | Beta (disponible públicamente) | | Autenticación | OAuth (Google/Microsoft) o API Key | | Documentación oficial | https://docs.fireflies.ai/ | | Blog de anuncio | https://fireflies.ai/blog/fireflies-mcp-server/ |

Herramientas disponibles vía MCP:

- `get_user` → Retorna información del perfil del usuario - `get_transcript` → Obtiene transcripción completa usando transcriptId - `get_transcripts` → Búsqueda en transcripciones pasadas con filtros

Configuración para Claude Desktop:

```json { "mcpServers": { "fireflies": { "command": "npx", "args": [ "mcp-remote", "https://api.fireflies.ai/mcp", "--header", "Authorization: Bearer YOUR_API_KEY_HERE" ] } } } ```

Fuentes verificadas: 1. https://fireflies.ai/blog/fireflies-mcp-server/ (blog oficial) 2. https://guide.fireflies.ai/articles/8272956938-learn-about-the-fireflies-mcp-server-model-context-protocol (documentación oficial) 3. https://docs.fireflies.ai/getting-started/introduction (API docs)

---

2️⃣ API de Fireflies.ai

✅ API GraphQL DISPONIBLE

Características principales:

- Tipo: GraphQL (no REST) - Endpoint único: Simplifica interacciones - Funcionalidades: - Upload de audio de reuniones - Fetch de transcripciones - Eliminación de transcripciones - Gestión de usuarios y roles - NLP personalizado para extracción de insights

Ventajas de GraphQL:

1. Precisión: Retorna exactamente lo solicitado (sin over-fetching) 2. Single endpoint: Más fácil de mantener 3. Flexibilidad: Queries evolucionan con necesidades del proyecto

Documentación completa: https://docs.fireflies.ai/
